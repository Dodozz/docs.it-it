# <a name="how-to-reference-com-objects-from-visual-basic"></a><span data-ttu-id="8738e-102">Procedura: Oggetti di riferimento COM da Visual Basic</span><span class="sxs-lookup"><span data-stu-id="8738e-102">How to: Reference COM Objects from Visual Basic</span></span>
<span data-ttu-id="8738e-103">In Visual Basic, l'aggiunta di riferimenti a oggetti COM con librerie dei tipi richiede la creazione di un assembly di interoperabilità per la libreria COM.</span><span class="sxs-lookup"><span data-stu-id="8738e-103">In Visual Basic, adding references to COM objects that have type libraries requires the creation of an interop assembly for the COM library.</span></span> <span data-ttu-id="8738e-104">I riferimenti ai membri dell'oggetto COM vengono indirizzati all'assembly di interoperabilità e quindi inoltrati all'oggetto COM effettivo.</span><span class="sxs-lookup"><span data-stu-id="8738e-104">References to the members of the COM object are routed to the interop assembly and then forwarded to the actual COM object.</span></span> <span data-ttu-id="8738e-105">Le risposte dall'oggetto COM vengono indirizzate all'assembly di interoperabilità e inoltrate al [!INCLUDE[dnprdnshort](~/includes/dnprdnshort-md.md)] dell'applicazione.</span><span class="sxs-lookup"><span data-stu-id="8738e-105">Responses from the COM object are routed to the interop assembly and forwarded to your [!INCLUDE[dnprdnshort](~/includes/dnprdnshort-md.md)] application.</span></span>  
  
 <span data-ttu-id="8738e-106">È possibile fare riferimento a un oggetto COM senza usare un assembly di interoperabilità per incorporare le informazioni sul tipo per l'oggetto COM in un assembly .NET.</span><span class="sxs-lookup"><span data-stu-id="8738e-106">You can reference a COM object without using an interop assembly by embedding the type information for the COM object in a .NET assembly.</span></span> <span data-ttu-id="8738e-107">Per incorporare le informazioni sul tipo, impostare il `Embed Interop Types` proprietà `True` per il riferimento all'oggetto COM.</span><span class="sxs-lookup"><span data-stu-id="8738e-107">To embed type information, set the `Embed Interop Types` property to `True` for the reference to the COM object.</span></span> <span data-ttu-id="8738e-108">Se esegue la compilazione usando il compilatore della riga di comando, usare il `/link` opzione per fare riferimento alla libreria COM.</span><span class="sxs-lookup"><span data-stu-id="8738e-108">If you are compiling by using the command-line compiler, use the `/link` option to reference the COM library.</span></span> <span data-ttu-id="8738e-109">Per altre informazioni, vedere [/link (Visual Basic)](../../../visual-basic/reference/command-line-compiler/link.md).</span><span class="sxs-lookup"><span data-stu-id="8738e-109">For more information, see [/link (Visual Basic)](../../../visual-basic/reference/command-line-compiler/link.md).</span></span>  
  
 <span data-ttu-id="8738e-110">Visual Basic crea automaticamente gli assembly di interoperabilità quando si aggiunge un riferimento a una libreria dei tipi dall'ambiente di sviluppo integrato (IDE).</span><span class="sxs-lookup"><span data-stu-id="8738e-110">Visual Basic automatically creates interop assemblies when you add a reference to a type library from the integrated development environment (IDE).</span></span> <span data-ttu-id="8738e-111">Quando si lavora dalla riga di comando, è possibile utilizzare l'utilità Tlbimp per creare manualmente gli assembly di interoperabilità.</span><span class="sxs-lookup"><span data-stu-id="8738e-111">When working from the command line, you can use the Tlbimp utility to manually create interop assemblies.</span></span>

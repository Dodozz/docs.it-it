 <span data-ttu-id="60925-110">Le variabili passate come argomenti `in` devono essere inizializzate prima di essere passate in una chiamata al metodo.</span><span class="sxs-lookup"><span data-stu-id="60925-110">Variables passed as `in` arguments must be initialized before being passed in a method call.</span></span> <span data-ttu-id="60925-111">Tuttavia, il metodo chiamato non può assegnare un valore o modificare l'argomento.</span><span class="sxs-lookup"><span data-stu-id="60925-111">However, the called method may not assign a value or modify the argument.</span></span>  
  
 <span data-ttu-id="60925-112">Nonostante le parole chiave `in`, `ref` e `out` determinino un comportamento differente in fase di esecuzione, non sono considerate parte della firma del metodo in fase di compilazione.</span><span class="sxs-lookup"><span data-stu-id="60925-112">Although the `in`, `ref`, and `out` keywords cause different run-time behavior, they are not considered part of the method signature at compile time.</span></span> <span data-ttu-id="60925-113">Non è quindi possibile eseguirne l'overload se l'unica differenza è che un metodo accetta un argomento `ref` o `in` e l'altro un argomento `out`.</span><span class="sxs-lookup"><span data-stu-id="60925-113">Therefore, methods cannot be overloaded if the only difference is that one method takes a `ref` or `in` argument and the other takes an `out` argument.</span></span> <span data-ttu-id="60925-114">Il codice seguente, ad esempio, non verrà compilato:</span><span class="sxs-lookup"><span data-stu-id="60925-114">The following code, for example, will not compile:</span></span>

<span data-ttu-id="60925-115">È consentito l'overload in base alla presenza di `in`:</span><span class="sxs-lookup"><span data-stu-id="60925-115">Overloading based on the presence of `in` is allowed:</span></span>

```csharp
class InOverloads

    public void SampleMethod(in int i) { }

```

## <a name="overload-resolution-rules"></a><span data-ttu-id="60925-116">Regole di risoluzione dell'overload</span><span class="sxs-lookup"><span data-stu-id="60925-116">Overload resolution rules</span></span>

<span data-ttu-id="60925-117">È possibile comprendere le regole di risoluzione dell'overload per i metodi con argomenti per valore rispetto ai metodi con argomenti `in` se si comprende la motivazione per gli argomenti `in`.</span><span class="sxs-lookup"><span data-stu-id="60925-117">You can understand the overload resolution rules for methods with by value vs. `in` arguments by understanding the motivation for `in` arguments.</span></span> <span data-ttu-id="60925-118">La definizione di metodi tramite parametri `in` costituisce un'ottimizzazione potenziale delle prestazioni.</span><span class="sxs-lookup"><span data-stu-id="60925-118">Defining methods using `in` parameters is a potential performance optimization.</span></span> <span data-ttu-id="60925-119">Alcuni argomenti di tipo `struct` possono essere di grandi dimensioni e quando vengono chiamati metodi all'interno di cicli ristretti o in percorsi di codice critici, il costo della copia di tali strutture ha una rilevanza fondamentale.</span><span class="sxs-lookup"><span data-stu-id="60925-119">Some `struct` type arguments may be large in size, and when methods are called in tight loops or critical code paths, the cost of copying those structures is critical.</span></span> <span data-ttu-id="60925-120">I metodi dichiarano parametri `in` per specificare che è possibile passare argomenti per riferimento in modo sicuro, perché il metodo chiamato non modifica lo stato degli argomenti.</span><span class="sxs-lookup"><span data-stu-id="60925-120">Methods declare `in` parameters to specify that arguments may be passed by reference safely because the called method does not modify the state of that argument.</span></span> <span data-ttu-id="60925-121">Il passaggio di tali argomenti per riferimento consente di evitare una copia potenzialmente dispendiosa.</span><span class="sxs-lookup"><span data-stu-id="60925-121">Passing those arguments by reference avoids the (potentially) expensive copy.</span></span> 

<span data-ttu-id="60925-122">Specificare `in` per gli argomenti presso il sito di chiamata è in genere facoltativo.</span><span class="sxs-lookup"><span data-stu-id="60925-122">Specifying `in` for arguments at the call site is typically optional.</span></span> <span data-ttu-id="60925-123">Non esiste alcuna differenza semantica tra il passaggio di argomenti per valore e il passaggio per riferimento tramite il modificatore `in`.</span><span class="sxs-lookup"><span data-stu-id="60925-123">There is no semantic difference between passing arguments by value and passing them by reference using the `in` modifier.</span></span> <span data-ttu-id="60925-124">Il modificatore `in` presso il sito di chiamata è facoltativo perché non è necessario indicare che il valore dell'argomento può essere modificato.</span><span class="sxs-lookup"><span data-stu-id="60925-124">The `in` modifier at the call site is optional because you don't need to indicate that the argument's value might be changed.</span></span> <span data-ttu-id="60925-125">Si aggiunge il modificatore `in` in modo esplicito presso il sito di chiamata per assicurarsi che l'argomento venga passato per riferimento, non per valore.</span><span class="sxs-lookup"><span data-stu-id="60925-125">You explicitly add the `in` modifier at the call site to ensure the argument is passed by reference, not by value.</span></span> <span data-ttu-id="60925-126">L'uso di `in` in modo esplicito ha i due effetti seguenti:</span><span class="sxs-lookup"><span data-stu-id="60925-126">Explicitly using `in` has the following two effects:</span></span>

<span data-ttu-id="60925-127">In primo luogo, se si specifica `in` presso il sito di chiamata si impone al compilatore di selezionare un metodo definito con un parametro `in` corrispondente.</span><span class="sxs-lookup"><span data-stu-id="60925-127">First, specifying `in` at the call site forces the compiler to select a method defined with a matching `in` parameter.</span></span> <span data-ttu-id="60925-128">In caso contrario, se due metodi si differenziano solo per la presenza di `in`, l'overload per valore rappresenta una corrispondenza migliore.</span><span class="sxs-lookup"><span data-stu-id="60925-128">Otherwise, when two methods differ only in the presence of `in`, the by value overload is a better match.</span></span>

<span data-ttu-id="60925-129">In secondo luogo, se si specifica `in` si dichiara l'intenzione di passare un argomento per riferimento.</span><span class="sxs-lookup"><span data-stu-id="60925-129">Second, specifying `in` declares your intent to pass an argument by reference.</span></span> <span data-ttu-id="60925-130">L'argomento usato con `in` deve rappresentare una posizione a cui sia possibile fare riferimento direttamente.</span><span class="sxs-lookup"><span data-stu-id="60925-130">The argument used with `in` must represent a location that can be directly referred to.</span></span> <span data-ttu-id="60925-131">Sono valide le stesse regole generali applicabili agli argomenti `out` e `ref`: non è possibile usare costanti, proprietà ordinarie o altre espressioni che producono valori.</span><span class="sxs-lookup"><span data-stu-id="60925-131">The same general rules for `out` and `ref` arguments apply: You cannot use constants, ordinary properties, or other expressions that produce values.</span></span> <span data-ttu-id="60925-132">In caso contrario, l'omissione di `in` presso il sito di chiamata informa il compilatore che è consentito creare una variabile temporanea da passare per riferimento di sola lettura al metodo.</span><span class="sxs-lookup"><span data-stu-id="60925-132">Otherwise, omitting `in` at the call site informs the compiler that you will allow it to create a temporary variable to pass by read-only reference to the method.</span></span> <span data-ttu-id="60925-133">Il compilatore crea una variabile temporanea per superare diverse restrizioni degli argomenti `in`:</span><span class="sxs-lookup"><span data-stu-id="60925-133">The compiler creates a temporary variable to overcome several restrictions with `in` arguments:</span></span>

- <span data-ttu-id="60925-134">Una variabile temporanea consente costanti in fase di compilazione come parametri `in`.</span><span class="sxs-lookup"><span data-stu-id="60925-134">A temporary variable allows compile-time constants as `in` parameters.</span></span>
- <span data-ttu-id="60925-135">Una variabile temporanea consente proprietà o altre espressioni per i parametri `in`.</span><span class="sxs-lookup"><span data-stu-id="60925-135">A temporary variable allows properties, or other expressions for `in` parameters.</span></span>
- <span data-ttu-id="60925-136">Una variabile temporanea consente argomenti che includono una conversione implicita dal tipo di argomento al tipo di parametro.</span><span class="sxs-lookup"><span data-stu-id="60925-136">A temporary variable allows arguments where there is an implicit conversion from the argument type to the parameter type.</span></span>

<span data-ttu-id="60925-137">In tutte le istanze precedenti, il compilatore crea una variabile temporanea che archivia il valore della costante, della proprietà o di un'altra espressione.</span><span class="sxs-lookup"><span data-stu-id="60925-137">In all the preceding instances, the compiler creates a temporary variable that stores the value of the constant, property, or other expression.</span></span>

<span data-ttu-id="60925-138">Il codice seguente illustra queste regole:</span><span class="sxs-lookup"><span data-stu-id="60925-138">The following code illustrates these rules:</span></span>

```csharp
static void Method(in int argument)
{
    // implementation removed
}

Method(5); // OK, temporary variable created.
Method(5L); // CS1503: no implicit conversion from long to int

Method(s); // OK, temporary int created with the value 0

Method(i); // passed by readonly reference
Method(in i); // passed by readonly reference, explicitly using `in`
```

<span data-ttu-id="60925-139">Si supponga a questo punto che sia disponibile un altro metodo che usa argomenti per valore.</span><span class="sxs-lookup"><span data-stu-id="60925-139">Now, suppose another method using by value arguments was available.</span></span> <span data-ttu-id="60925-140">I risultati cambiano, come illustrato nel codice seguente:</span><span class="sxs-lookup"><span data-stu-id="60925-140">The results change as shown in the following code:</span></span>

```csharp
static void Method(int argument)
{
    // implementation removed
}

static void Method(in int argument)
{
    // implementation removed
}

Method(5); // Calls overload passed by value
Method(5L); // CS1503: no implicit conversion from long to int
short s = 0;
Method(s); // Calls overload passed by value.
Method(in s); // CS1503: cannot convert from in short to in int
int i = 42;
Method(i); // Calls overload passed by value
Method(in i); // passed by readonly reference, explicitly using `in`
```

<span data-ttu-id="60925-141">L'unica chiamata a un metodo in cui l'argomento viene passato per riferimento è quella finale.</span><span class="sxs-lookup"><span data-stu-id="60925-141">The only method call where the argument is passed by reference is the final one.</span></span>

> [!NOTE]

<span data-ttu-id="a0e9e-106">La resilienza è la capacità di correggere gli errori e continuare a funzionare.</span><span class="sxs-lookup"><span data-stu-id="a0e9e-106">Resiliency is the ability to recover from failures and continue to function.</span></span> <span data-ttu-id="a0e9e-107">La resilienza non è presente su come evitare errori, ma accettare il fatto che si verificheranno errori e quindi rispondere in modo da evitare tempi di inattività o perdita di dati.</span><span class="sxs-lookup"><span data-stu-id="a0e9e-107">Resiliency is not about avoiding failures, but accepting the fact that failures will occur, and then responding to them in a way that avoids downtime or data loss.</span></span> <span data-ttu-id="a0e9e-108">L'obiettivo della resilienza è riportare l'applicazione in uno stato pienamente operativo dopo un errore.</span><span class="sxs-lookup"><span data-stu-id="a0e9e-108">The goal of resiliency is to return the application to a fully functioning state after a failure.</span></span>

## <a name="handling-partial-failure"></a><span data-ttu-id="a0e9e-114">Gestione degli errori parziali</span><span class="sxs-lookup"><span data-stu-id="a0e9e-114">Handling partial failure</span></span>

<span data-ttu-id="a0e9e-115">In un'applicazione basata su cloud, è presente un onnipresente rischio di errori parziali.</span><span class="sxs-lookup"><span data-stu-id="a0e9e-115">In a cloud-based application, there's an ever-present risk of partial failure.</span></span> <span data-ttu-id="a0e9e-116">Ad esempio, un'istanza del singolo sito Web o in un contenitore potrebbe non riuscire oppure potrebbe essere non disponibile o non risponda per breve tempo.</span><span class="sxs-lookup"><span data-stu-id="a0e9e-116">For instance, a single website instance or a container might fail, or it might be unavailable or unresponsive for a short time.</span></span> <span data-ttu-id="a0e9e-117">In alternativa, una singola macchina virtuale o server potrebbe arrestarsi in modo anomalo.</span><span class="sxs-lookup"><span data-stu-id="a0e9e-117">Or, a single VM or server might crash.</span></span>

<span data-ttu-id="a0e9e-120">Ad esempio, prendere in considerazione un'applicazione monolitica di .NET che accede a un database nel Database SQL di Azure.</span><span class="sxs-lookup"><span data-stu-id="a0e9e-120">For example, consider a monolithic .NET application that accesses a database in Azure SQL Database.</span></span> <span data-ttu-id="a0e9e-121">Se il database SQL di Azure o qualsiasi altro servizio di terze parti non risponde per una breve tempo (un database SQL di Azure potrebbe essere spostato in un server o un altro nodo e da non rispondere per pochi secondi), quando l'utente tenta di eseguire qualsiasi azione, l'applicazione potrebbe bloccarsi e Mostra w un'eccezione nello stesso momento.</span><span class="sxs-lookup"><span data-stu-id="a0e9e-121">If the Azure SQL database or any other third-party service is unresponsive for a brief time (an Azure SQL database might be moved to a different node or server, and be unresponsive for a few seconds), when the user tries to do any action, the application might crash and show an exception at the same moment.</span></span>

<span data-ttu-id="a0e9e-122">Uno scenario simile può verificarsi in un'app che utilizza i servizi HTTP.</span><span class="sxs-lookup"><span data-stu-id="a0e9e-122">A similar scenario might occur in an app that consumes HTTP services.</span></span> <span data-ttu-id="a0e9e-123">La rete o il servizio stesso non sia disponibile nel cloud in caso di errore breve e temporaneo.</span><span class="sxs-lookup"><span data-stu-id="a0e9e-123">The network or the service itself might not be available in the cloud during a short, transient failure.</span></span>

<span data-ttu-id="a0e9e-124">Un'applicazione resiliente simile a quello illustrato nella figura 4-9 deve implementare tecniche quali "tentativi con backoff esponenziale" per assegnare all'applicazione la possibilità di gestire gli errori temporanei nelle risorse.</span><span class="sxs-lookup"><span data-stu-id="a0e9e-124">A resilient application like the one shown in Figure 4-9 should implement techniques like "retries with exponential backoff" to give the application an opportunity to handle transient failures in resources.</span></span> <span data-ttu-id="a0e9e-125">È anche consigliabile usare "gli interruttori di circuito" nelle applicazioni.</span><span class="sxs-lookup"><span data-stu-id="a0e9e-125">You also should use "circuit breakers" in your applications.</span></span> <span data-ttu-id="a0e9e-126">Un interruttore di circuito si arresta un'applicazione tenti di accedere a una risorsa quando è effettivamente un errore a lungo termine.</span><span class="sxs-lookup"><span data-stu-id="a0e9e-126">A circuit breaker stops an application from trying to access a resource when it's actually a long-term failure.</span></span> <span data-ttu-id="a0e9e-127">Usando un interruttore di circuito, l'applicazione evita provocato un attacco denial of service a se stesso.</span><span class="sxs-lookup"><span data-stu-id="a0e9e-127">By using a circuit breaker, the application avoids provoking a denial of service to itself.</span></span>

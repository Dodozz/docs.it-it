<span data-ttu-id="bf4c7-252">I servizi con e senza stato sono complementari.</span><span class="sxs-lookup"><span data-stu-id="bf4c7-252">Stateless and stateful services are complementary.</span></span> <span data-ttu-id="bf4c7-253">Ad esempio, un servizio con stato può essere suddivisi in più partizioni.</span><span class="sxs-lookup"><span data-stu-id="bf4c7-253">For instance, a stateful service could be split into multiple partitions.</span></span> <span data-ttu-id="bf4c7-254">Per accedere a tali partizioni, potrebbe essere necessario un servizio senza stato che agisca come servizio gateway che sappia come risolvere ogni partizione in base alle chiavi di partizione.</span><span class="sxs-lookup"><span data-stu-id="bf4c7-254">To access those partitions, you might need a stateless service acting as a gateway service that knows how to address each partition based on partition keys.</span></span>

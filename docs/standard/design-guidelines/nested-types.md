# <a name="nested-types"></a><span data-ttu-id="ff1ec-102">Tipi annidati</span><span class="sxs-lookup"><span data-stu-id="ff1ec-102">Nested Types</span></span>
<span data-ttu-id="ff1ec-103">Un tipo annidato è un tipo definito all'interno dell'ambito di un altro tipo, che viene chiamato il tipo di inclusione.</span><span class="sxs-lookup"><span data-stu-id="ff1ec-103">A nested type is a type defined within the scope of another type, which is called the enclosing type.</span></span> <span data-ttu-id="ff1ec-104">Un tipo annidato può accedere a tutti i membri del tipo di inclusione.</span><span class="sxs-lookup"><span data-stu-id="ff1ec-104">A nested type has access to all members of its enclosing type.</span></span> <span data-ttu-id="ff1ec-105">Ad esempio, ha accesso ai campi privati definiti nel tipo di inclusione e proteggere i campi definiti in tutti i predecessori del tipo di inclusione.</span><span class="sxs-lookup"><span data-stu-id="ff1ec-105">For example, it has access to private fields defined in the enclosing type and to protected fields defined in all ascendants of the enclosing type.</span></span>  
  
 <span data-ttu-id="ff1ec-106">In generale, i tipi annidati devono essere utilizzati solo se necessario.</span><span class="sxs-lookup"><span data-stu-id="ff1ec-106">In general, nested types should be used sparingly.</span></span> <span data-ttu-id="ff1ec-107">Le ragioni sono molteplici.</span><span class="sxs-lookup"><span data-stu-id="ff1ec-107">There are several reasons for this.</span></span> <span data-ttu-id="ff1ec-108">Alcuni sviluppatori non sono completamente dimestichezza con il concetto.</span><span class="sxs-lookup"><span data-stu-id="ff1ec-108">Some developers are not fully familiar with the concept.</span></span> <span data-ttu-id="ff1ec-109">Questi sviluppatori potrebbero, ad esempio, avere problemi con la sintassi di dichiarazione di variabili di tipi annidati.</span><span class="sxs-lookup"><span data-stu-id="ff1ec-109">These developers might, for example, have problems with the syntax of declaring variables of nested types.</span></span> <span data-ttu-id="ff1ec-110">I tipi annidati sono anche molto strettamente collegati i relativi tipi di inclusione e di conseguenza non sono adatte per essere tipi di uso generico.</span><span class="sxs-lookup"><span data-stu-id="ff1ec-110">Nested types are also very tightly coupled with their enclosing types, and as such are not suited to be general-purpose types.</span></span>  
  
 <span data-ttu-id="ff1ec-111">I tipi annidati sono ideali per i dettagli di implementazione dei relativi tipi di inclusione di modellazione.</span><span class="sxs-lookup"><span data-stu-id="ff1ec-111">Nested types are best suited for modeling implementation details of their enclosing types.</span></span> <span data-ttu-id="ff1ec-112">L'utente finale deve avere raramente per dichiarare le variabili di un tipo annidato e quasi mai necessario creare un'istanza esplicita di tipi annidati.</span><span class="sxs-lookup"><span data-stu-id="ff1ec-112">The end user should rarely have to declare variables of a nested type and almost never should have to explicitly instantiate nested types.</span></span> <span data-ttu-id="ff1ec-113">Ad esempio, l'enumeratore di una raccolta può essere un tipo annidato di tale raccolta.</span><span class="sxs-lookup"><span data-stu-id="ff1ec-113">For example, the enumerator of a collection can be a nested type of that collection.</span></span> <span data-ttu-id="ff1ec-114">Gli enumeratori sono in genere creata un'istanza in base al tipo di inclusione e perché molti linguaggi supportano l'istruzione foreach, le variabili di enumeratore raramente devono essere dichiarati dall'utente finale.</span><span class="sxs-lookup"><span data-stu-id="ff1ec-114">Enumerators are usually instantiated by their enclosing type, and because many languages support the foreach statement, enumerator variables rarely have to be declared by the end user.</span></span>  
  
 <span data-ttu-id="ff1ec-115">**✓ DO** utilizzare tipi annidati quando la relazione tra il tipo annidato e il relativo tipo esterno è tale che la semantica di accessibilità del membro auspicabile.</span><span class="sxs-lookup"><span data-stu-id="ff1ec-115">**✓ DO** use nested types when the relationship between the nested type and its outer type is such that member-accessibility semantics are desirable.</span></span>  
  
 <span data-ttu-id="ff1ec-116">**X DO NOT** utilizzare tipi annidati pubblici come un raggruppamento logico costruire; utilizzare gli spazi dei nomi per questo oggetto.</span><span class="sxs-lookup"><span data-stu-id="ff1ec-116">**X DO NOT** use public nested types as a logical grouping construct; use namespaces for this.</span></span>  
  
 <span data-ttu-id="ff1ec-117">**X AVOID** esposte pubblicamente tipi annidati.</span><span class="sxs-lookup"><span data-stu-id="ff1ec-117">**X AVOID** publicly exposed nested types.</span></span> <span data-ttu-id="ff1ec-118">L'unica eccezione è se le variabili del tipo annidato devono essere dichiarati solo in rari casi, ad esempio la creazione di sottoclassi o altri scenari di personalizzazione avanzate.</span><span class="sxs-lookup"><span data-stu-id="ff1ec-118">The only exception to this is if variables of the nested type need to be declared only in rare scenarios such as subclassing or other advanced customization scenarios.</span></span>  
  
 <span data-ttu-id="ff1ec-119">**X DO NOT** utilizzare tipi annidati se il tipo è probabilmente necessario fare riferimento all'esterno del tipo che lo contiene.</span><span class="sxs-lookup"><span data-stu-id="ff1ec-119">**X DO NOT** use nested types if the type is likely to be referenced outside of the containing type.</span></span>  
  
 <span data-ttu-id="ff1ec-120">Ad esempio, un enum passato a un metodo definito in una classe non deve essere definito come un tipo annidato nella classe.</span><span class="sxs-lookup"><span data-stu-id="ff1ec-120">For example, an enum passed to a method defined on a class should not be defined as a nested type in the class.</span></span>  
  
 <span data-ttu-id="ff1ec-121">**X DO NOT** utilizzare tipi annidati se devono essere creata un'istanza mediante il codice client.</span><span class="sxs-lookup"><span data-stu-id="ff1ec-121">**X DO NOT** use nested types if they need to be instantiated by client code.</span></span>  <span data-ttu-id="ff1ec-122">Se un tipo ha un costruttore pubblico, deve probabilmente non essere stato annidato.</span><span class="sxs-lookup"><span data-stu-id="ff1ec-122">If a type has a public constructor, it should probably not be nested.</span></span>  
  
 <span data-ttu-id="ff1ec-123">Se è possibile creare istanze di un tipo, che sembrano indicare il tipo ha una posizione nel framework di per sé (è possibile crearlo, usarlo ed eliminarla definitivamente senza mai usare il tipo esterno) e pertanto non devono essere nidificate.</span><span class="sxs-lookup"><span data-stu-id="ff1ec-123">If a type can be instantiated, that seems to indicate the type has a place in the framework on its own (you can create it, work with it, and destroy it without ever using the outer type), and thus should not be nested.</span></span> <span data-ttu-id="ff1ec-124">Tipi interni non devono essere riutilizzati ampiamente all'esterno del tipo outer senza alcuna relazione alcun modo per il tipo esterno.</span><span class="sxs-lookup"><span data-stu-id="ff1ec-124">Inner types should not be widely reused outside of the outer type without any relationship whatsoever to the outer type.</span></span>  
  
 <span data-ttu-id="ff1ec-125">**X DO NOT** definire un tipo annidato come membro di interfaccia.</span><span class="sxs-lookup"><span data-stu-id="ff1ec-125">**X DO NOT** define a nested type as a member of an interface.</span></span> <span data-ttu-id="ff1ec-126">Molti linguaggi non supportano tale costrutto.</span><span class="sxs-lookup"><span data-stu-id="ff1ec-126">Many languages do not support such a construct.</span></span>

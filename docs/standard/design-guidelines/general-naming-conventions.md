# <a name="general-naming-conventions"></a><span data-ttu-id="cf3ec-102">Convenzioni di denominazione generali</span><span class="sxs-lookup"><span data-stu-id="cf3ec-102">General Naming Conventions</span></span>
<span data-ttu-id="cf3ec-103">Questa sezione descrive convenzioni di denominazione generali correlate a scelta delle parole, le linee guida sull'uso di abbreviazioni e gli acronimi e consigli su come evitare di utilizzare nomi specifici del linguaggio.</span><span class="sxs-lookup"><span data-stu-id="cf3ec-103">This section describes general naming conventions that relate to word choice, guidelines on using abbreviations and acronyms, and recommendations on how to avoid using language-specific names.</span></span>  
  
## <a name="word-choice"></a><span data-ttu-id="cf3ec-104">Scelta delle parole</span><span class="sxs-lookup"><span data-stu-id="cf3ec-104">Word Choice</span></span>  
 <span data-ttu-id="cf3ec-105">**✓ DO** scegliere i nomi degli identificatori facilmente leggibili.</span><span class="sxs-lookup"><span data-stu-id="cf3ec-105">**✓ DO** choose easily readable identifier names.</span></span>  
  
 <span data-ttu-id="cf3ec-106">Ad esempio, una proprietà denominata `HorizontalAlignment` risulta più inglese-leggibile rispetto a `AlignmentHorizontal`.</span><span class="sxs-lookup"><span data-stu-id="cf3ec-106">For example, a property named `HorizontalAlignment` is more English-readable than `AlignmentHorizontal`.</span></span>  
  
 <span data-ttu-id="cf3ec-107">**✓ DO** privilegiare la leggibilità ragioni di brevità.</span><span class="sxs-lookup"><span data-stu-id="cf3ec-107">**✓ DO** favor readability over brevity.</span></span>  
  
 <span data-ttu-id="cf3ec-108">Il nome della proprietà `CanScrollHorizontally` migliore `ScrollableX` (un riferimento all'asse x).</span><span class="sxs-lookup"><span data-stu-id="cf3ec-108">The property name `CanScrollHorizontally` is better than `ScrollableX` (an obscure reference to the X-axis).</span></span>  
  
 <span data-ttu-id="cf3ec-109">**X DO NOT** utilizzare caratteri di sottolineatura, trattini o altri caratteri non alfanumerici.</span><span class="sxs-lookup"><span data-stu-id="cf3ec-109">**X DO NOT** use underscores, hyphens, or any other nonalphanumeric characters.</span></span>  
  
 <span data-ttu-id="cf3ec-110">**X DO NOT** utilizzare la notazione ungherese.</span><span class="sxs-lookup"><span data-stu-id="cf3ec-110">**X DO NOT** use Hungarian notation.</span></span>  
  
 <span data-ttu-id="cf3ec-111">**X AVOID** utilizzando gli identificatori che sono in conflitto con le parole chiave di ampiamente utilizzato linguaggi di programmazione.</span><span class="sxs-lookup"><span data-stu-id="cf3ec-111">**X AVOID** using identifiers that conflict with keywords of widely used programming languages.</span></span>  
  
 <span data-ttu-id="cf3ec-112">In base alla regola 4 della specifica CLS (Common Language), tutti i linguaggi conformi a devono fornire un meccanismo che consente l'accesso agli elementi denominati che usano una parola chiave del linguaggio come identificatore.</span><span class="sxs-lookup"><span data-stu-id="cf3ec-112">According to Rule 4 of the Common Language Specification (CLS), all compliant languages must provide a mechanism that allows access to named items that use a keyword of that language as an identifier.</span></span> <span data-ttu-id="cf3ec-113">C#, ad esempio, Usa il simbolo come meccanismo di escape in questo caso @.</span><span class="sxs-lookup"><span data-stu-id="cf3ec-113">C#, for example, uses the @ sign as an escape mechanism in this case.</span></span> <span data-ttu-id="cf3ec-114">Tuttavia, è comunque consigliabile evitare le parole chiave comuni in quanto è molto più difficile da usare un metodo con la sequenza di escape rispetto a uno senza di essa.</span><span class="sxs-lookup"><span data-stu-id="cf3ec-114">However, it is still a good idea to avoid common keywords because it is much more difficult to use a method with the escape sequence than one without it.</span></span>  
  
## <a name="using-abbreviations-and-acronyms"></a><span data-ttu-id="cf3ec-115">Usando gli acronimi e abbreviazioni</span><span class="sxs-lookup"><span data-stu-id="cf3ec-115">Using Abbreviations and Acronyms</span></span>  
 <span data-ttu-id="cf3ec-116">**X DO NOT** utilizzare abbreviazioni o termini più semplici come parte dei nomi degli identificatori.</span><span class="sxs-lookup"><span data-stu-id="cf3ec-116">**X DO NOT** use abbreviations or contractions as part of identifier names.</span></span>  
  
 <span data-ttu-id="cf3ec-117">Ad esempio, usare `GetWindow` anziché `GetWin`.</span><span class="sxs-lookup"><span data-stu-id="cf3ec-117">For example, use `GetWindow` rather than `GetWin`.</span></span>  
  
 <span data-ttu-id="cf3ec-118">**X DO NOT** utilizzare acronimi che non sono molto diffusa e anche se sono solo quando necessario.</span><span class="sxs-lookup"><span data-stu-id="cf3ec-118">**X DO NOT** use any acronyms that are not widely accepted, and even if they are, only when necessary.</span></span>  
  
## <a name="avoiding-language-specific-names"></a><span data-ttu-id="cf3ec-119">Evitare nomi specifici del linguaggio</span><span class="sxs-lookup"><span data-stu-id="cf3ec-119">Avoiding Language-Specific Names</span></span>  
 <span data-ttu-id="cf3ec-120">**✓ DO** utilizzare nomi significativi anziché le parole chiave specifiche della lingua per i nomi dei tipi.</span><span class="sxs-lookup"><span data-stu-id="cf3ec-120">**✓ DO** use semantically interesting names rather than language-specific keywords for type names.</span></span>  
  
 <span data-ttu-id="cf3ec-121">Ad esempio, `GetLength` è un nome migliore rispetto a `GetInt`.</span><span class="sxs-lookup"><span data-stu-id="cf3ec-121">For example, `GetLength` is a better name than `GetInt`.</span></span>  
  
 <span data-ttu-id="cf3ec-122">**✓ DO** usare un nome di tipo generico di CLR, anziché un nome specifico della lingua, in rari casi in cui un identificatore non ha alcun significato semantico oltre il relativo tipo.</span><span class="sxs-lookup"><span data-stu-id="cf3ec-122">**✓ DO** use a generic CLR type name, rather than a language-specific name, in the rare cases when an identifier has no semantic meaning beyond its type.</span></span>

## <a name="naming-new-versions-of-existing-apis"></a><span data-ttu-id="cf3ec-186">Denominazione di nuove versioni delle API esistente</span><span class="sxs-lookup"><span data-stu-id="cf3ec-186">Naming New Versions of Existing APIs</span></span>  
 <span data-ttu-id="cf3ec-187">**✓ DO** utilizzare un nome simile all'API precedente durante la creazione di nuove versioni di un'API esistente.</span><span class="sxs-lookup"><span data-stu-id="cf3ec-187">**✓ DO** use a name similar to the old API when creating new versions of an existing API.</span></span>  
  
 <span data-ttu-id="cf3ec-188">Ciò consente di evidenziare la relazione tra le API.</span><span class="sxs-lookup"><span data-stu-id="cf3ec-188">This helps to highlight the relationship between the APIs.</span></span>  
  
 <span data-ttu-id="cf3ec-189">**✓ DO** preferisce aggiunta di un suffisso anziché un prefisso per indicare una nuova versione di un'API esistente.</span><span class="sxs-lookup"><span data-stu-id="cf3ec-189">**✓ DO** prefer adding a suffix rather than a prefix to indicate a new version of an existing API.</span></span>  
  
 <span data-ttu-id="cf3ec-190">Questo aiuterà individuazione quando si Esplora la documentazione, o usando IntelliSense.</span><span class="sxs-lookup"><span data-stu-id="cf3ec-190">This will assist discovery when browsing documentation, or using IntelliSense.</span></span> <span data-ttu-id="cf3ec-191">La versione precedente dell'API sarà organizzata vicino le nuove API, perché la maggior parte dei browser e IntelliSense visualizza gli identificatori in ordine alfabetico.</span><span class="sxs-lookup"><span data-stu-id="cf3ec-191">The old version of the API will be organized close to the new APIs, because most browsers and IntelliSense show identifiers in alphabetical order.</span></span>

 <span data-ttu-id="cf3ec-193">**✓ DO** utilizzare un suffisso numerico per indicare una nuova versione di un'API esistente, in particolare se il nome dell'API esistente è l'unico nome appropriato (ad esempio, se è uno standard del settore) e se si aggiungono qualsiasi significativo suffisso (o la modifica del nome) non è un'app opzione ropriate.</span><span class="sxs-lookup"><span data-stu-id="cf3ec-193">**✓ DO** use a numeric suffix to indicate a new version of an existing API, particularly if the existing name of the API is the only name that makes sense (i.e., if it is an industry standard) and if adding any meaningful suffix (or changing the name) is not an appropriate option.</span></span>  
  
 <span data-ttu-id="cf3ec-194">**X DO NOT** utilizzare "Ex" (o una simile) suffisso di un identificatore per distinguerlo da una versione precedente dell'API stessa.</span><span class="sxs-lookup"><span data-stu-id="cf3ec-194">**X DO NOT** use the "Ex" (or a similar) suffix for an identifier to distinguish it from an earlier version of the same API.</span></span>  
  
 <span data-ttu-id="cf3ec-195">**✓ DO** utilizzano il suffisso "64" quando si introduce le versioni delle API che operano su un valore integer a 64 bit (un valore long integer) anziché un intero a 32 bit.</span><span class="sxs-lookup"><span data-stu-id="cf3ec-195">**✓ DO** use the "64" suffix when introducing versions of APIs that operate on a 64-bit integer (a long integer) instead of a 32-bit integer.</span></span> <span data-ttu-id="cf3ec-196">È sufficiente adottare questo approccio quando è presente l'API di 32 bit esistente. non eseguire questa operazione per la nuove API con solo una versione a 64 bit.</span><span class="sxs-lookup"><span data-stu-id="cf3ec-196">You only need to take this approach when the existing 32-bit API exists; don’t do it for brand new APIs with only a 64-bit version.</span></span>

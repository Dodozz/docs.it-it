## <a name="container-design-principles"></a><span data-ttu-id="9045d-112">Principi di progettazione dei contenitori</span><span class="sxs-lookup"><span data-stu-id="9045d-112">Container design principles</span></span>

<span data-ttu-id="9045d-113">Nel modello di contenitore, un'istanza dell'immagine del contenitore rappresenta un singolo processo.</span><span class="sxs-lookup"><span data-stu-id="9045d-113">In the container model, a container image instance represents a single process.</span></span> <span data-ttu-id="9045d-114">Definendo l'immagine di un contenitore come limite del processo, è possibile creare le primitive che possono essere usate per ridimensionare il processo o suddividerlo in batch.</span><span class="sxs-lookup"><span data-stu-id="9045d-114">By defining a container image as a process boundary, you can create primitives that can be used to scale the process or to batch it.</span></span>

<span data-ttu-id="9045d-115">Quando si progetta un'immagine del contenitore, Dockerfile conterrà una definizione [ENTRYPOINT](https://docs.docker.com/engine/reference/builder/).</span><span class="sxs-lookup"><span data-stu-id="9045d-115">When you design a container image, you will see an [ENTRYPOINT](https://docs.docker.com/engine/reference/builder/) definition in the Dockerfile.</span></span> <span data-ttu-id="9045d-116">In questo modo viene definito il processo il cui ciclo di vita controlla il ciclo di vita del contenitore.</span><span class="sxs-lookup"><span data-stu-id="9045d-116">This defines the process whose lifetime controls the lifetime of the container.</span></span> <span data-ttu-id="9045d-117">Al completamento del processo, termina il ciclo di vita del contenitore.</span><span class="sxs-lookup"><span data-stu-id="9045d-117">When the process completes, the container lifecycle ends.</span></span> <span data-ttu-id="9045d-118">I contenitori potrebbero rappresentare processi a esecuzione prolungata come i server Web, ma possono anche rappresentare i processi di breve durata, come i processi batch, che in precedenza potrebbero essere stati implementati come [processi Web](https://docs.microsoft.com/azure/app-service-web/websites-webjobs-resources) di Azure.</span><span class="sxs-lookup"><span data-stu-id="9045d-118">Containers might represent long-running processes like web servers, but can also represent short-lived processes like batch jobs, which formerly might have been implemented as Azure [WebJobs](https://docs.microsoft.com/azure/app-service-web/websites-webjobs-resources).</span></span>

<span data-ttu-id="9045d-119">Se il processo ha esito negativo, il ciclo di vita del contenitore termina e subentra l'agente di orchestrazione.</span><span class="sxs-lookup"><span data-stu-id="9045d-119">If the process fails, the container ends, and the orchestrator takes over.</span></span> <span data-ttu-id="9045d-120">Se l'agente di orchestrazione è stato configurato per mantenere cinque istanze in esecuzione e una non riesce, crea un'altra istanza del contenitore per sostituire il processo non riuscito.</span><span class="sxs-lookup"><span data-stu-id="9045d-120">If the orchestrator was configured to keep five instances running and one fails, the orchestrator will create another container instance to replace the failed process.</span></span> <span data-ttu-id="9045d-121">In un processo batch, il processo viene avviato con parametri.</span><span class="sxs-lookup"><span data-stu-id="9045d-121">In a batch job, the process is started with parameters.</span></span> <span data-ttu-id="9045d-122">Al termine del processo, il lavoro risulta completato.</span><span class="sxs-lookup"><span data-stu-id="9045d-122">When the process completes, the work is complete.</span></span> <span data-ttu-id="9045d-123">Più avanti in questo materiale sussidiario viene eseguito il drill-down degli agenti di orchestrazione.</span><span class="sxs-lookup"><span data-stu-id="9045d-123">This guidance drills-down on orchestrators, later on.</span></span>

<span data-ttu-id="9045d-124">In alcune situazioni occorre che più processi vengano eseguiti in un singolo contenitore.</span><span class="sxs-lookup"><span data-stu-id="9045d-124">You might find a scenario where you want multiple processes running in a single container.</span></span> <span data-ttu-id="9045d-125">In questi casi, poiché può esserci un solo punto di ingresso per ogni contenitore, è possibile eseguire uno script all'interno del contenitore per avviare tutti i programmi necessari.</span><span class="sxs-lookup"><span data-stu-id="9045d-125">For that scenario, since there can be only one entry point per container, you could run a script within the container that launches as many programs as needed.</span></span> <span data-ttu-id="9045d-126">È ad esempio possibile usare [Supervisor](http://supervisord.org/) o uno strumento simile per avviare più processi in un solo contenitore.</span><span class="sxs-lookup"><span data-stu-id="9045d-126">For example, you can use [Supervisor](http://supervisord.org/) or a similar tool to take care of launching multiple processes inside a single container.</span></span> <span data-ttu-id="9045d-127">Anche se è possibile trovare architetture che supportano più processi per ogni contenitore, questo approccio non è tuttavia molto comune.</span><span class="sxs-lookup"><span data-stu-id="9045d-127">However, even though you can find architectures that hold multiple processes per container, that approach is not very common.</span></span>

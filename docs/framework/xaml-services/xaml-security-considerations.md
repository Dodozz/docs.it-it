 <span data-ttu-id="51b04-118">L'architettura di servizi XAML di .NET Framework per lettori XAML e writer XAML spesso richiede la condivisione di un lettore XAML per un writer XAML, o un contesto dello schema XAML condiviso.</span><span class="sxs-lookup"><span data-stu-id="51b04-118">The .NET Framework XAML Services architecture for XAML readers and XAML writers often requires sharing a XAML reader to a XAML writer, or a shared XAML schema context.</span></span> <span data-ttu-id="51b04-119">La condivisione di oggetti o contesti potrebbe essere necessaria se si sta scrivendo la logica di ciclo di nodi XAML o fornendo un oggetto personalizzato di percorso di salvataggio.</span><span class="sxs-lookup"><span data-stu-id="51b04-119">Sharing objects or contexts might be required if you are writing XAML node loop logic, or providing a custom save path.</span></span> <span data-ttu-id="51b04-120">Non si devono condividere le istanze del reader XAML, il contesto dello schema XAML non predefinito o le impostazioni per le classi di reader e writer XAML tra il codice attendibile e non attendibile.</span><span class="sxs-lookup"><span data-stu-id="51b04-120">You should not share XAML reader instances, nondefault XAML schema context, or settings for XAML reader/writer classes between trusted and untrusted code.</span></span>  
  
 <span data-ttu-id="51b04-121">La maggior parte degli scenari e le operazioni che interessano oggetto XAML scrittura per un tipo basato su Common Language Runtime supporta sufficiente utilizzare il contesto dello schema XAML predefinito.</span><span class="sxs-lookup"><span data-stu-id="51b04-121">Most scenarios and operations involving XAML object writing for a CLR-based type backing can just use default XAML schema context.</span></span> <span data-ttu-id="51b04-122">In modo esplicito il contesto dello schema XAML predefinito non include impostazioni che potrebbero compromettere l'attendibilità totale.</span><span class="sxs-lookup"><span data-stu-id="51b04-122">The default XAML schema context does not explicitly include settings that could compromise full trust.</span></span> <span data-ttu-id="51b04-123">In questo modo è possibile condividere contesto tra attendibili e componenti di reader e writer XAML.</span><span class="sxs-lookup"><span data-stu-id="51b04-123">It is thus safe to share context between trusted and untrusted XAML reader/writer components.</span></span> <span data-ttu-id="51b04-124">Tuttavia, se in questo caso, è comunque consigliabile tenere separati questi lettori e writer <xref:System.AppDomain> ambiti, con uno di essi in particolare previsto/sandbox per un'attendibilità parziale.</span><span class="sxs-lookup"><span data-stu-id="51b04-124">However, if you do this, it is still a best practice to keep such readers and writers in separate <xref:System.AppDomain> scopes, with one of them specifically intended/sandboxed for partial trust.</span></span>

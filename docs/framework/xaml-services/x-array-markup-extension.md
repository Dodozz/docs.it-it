## <a name="remarks"></a><span data-ttu-id="03bc6-112">Note</span><span class="sxs-lookup"><span data-stu-id="03bc6-112">Remarks</span></span>

 <span data-ttu-id="03bc6-115">Nell'implementazione dei servizi XAML di .NET Framework, la relazione tra il tipo XAML di input e output CLR <xref:System.Type> della matrice creata è influenzata dal contesto del servizio per le estensioni di markup.</span><span class="sxs-lookup"><span data-stu-id="03bc6-115">In the .NET Framework XAML Services implementation, the relationship between the input XAML type and the output CLR <xref:System.Type> of the created array is influenced by service context for markup extensions.</span></span> <span data-ttu-id="03bc6-116">L'output <xref:System.Type> è il <xref:System.Xaml.XamlType.UnderlyingType%2A> del tipo XAML di input, dopo aver cercato le necessarie <xref:System.Xaml.XamlType> basato sul contesto dello schema XAML e <xref:System.Windows.Markup.IXamlTypeResolver> service fornisce il contesto.</span><span class="sxs-lookup"><span data-stu-id="03bc6-116">The output <xref:System.Type> is the <xref:System.Xaml.XamlType.UnderlyingType%2A> of the input XAML type, after looking up the necessary <xref:System.Xaml.XamlType> based on XAML schema context and the <xref:System.Windows.Markup.IXamlTypeResolver> service the context provides.</span></span>  
  
 <span data-ttu-id="03bc6-117">Durante l'elaborazione, il contenuto della matrice è assegnato al `ArrayExtension.Items` proprietà intrinseca.</span><span class="sxs-lookup"><span data-stu-id="03bc6-117">When processed, the array contents are assigned to the `ArrayExtension.Items` intrinsic property.</span></span> <span data-ttu-id="03bc6-118">Nel <xref:System.Windows.Markup.ArrayExtension> implementazione, è rappresentato dal <xref:System.Windows.Markup.ArrayExtension.Items%2A?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="03bc6-118">In the <xref:System.Windows.Markup.ArrayExtension> implementation, this is represented by <xref:System.Windows.Markup.ArrayExtension.Items%2A?displayProperty=nameWithType>.</span></span>  
  
 <span data-ttu-id="03bc6-119">Nell'implementazione di servizi XAML di .NET Framework, la gestione di questa estensione di markup viene definita per il <xref:System.Windows.Markup.ArrayExtension> classe.</span><span class="sxs-lookup"><span data-stu-id="03bc6-119">In the .NET Framework XAML Services implementation, the handling for this markup extension is defined by the <xref:System.Windows.Markup.ArrayExtension> class.</span></span> <span data-ttu-id="03bc6-120"><xref:System.Windows.Markup.ArrayExtension> non è bloccato e può essere usato come base per un'implementazione dell'estensione di markup per un tipo di matrice personalizzata.</span><span class="sxs-lookup"><span data-stu-id="03bc6-120"><xref:System.Windows.Markup.ArrayExtension> is not sealed, and could be used as the basis for a markup extension implementation for a custom array type.</span></span>  
  
 <span data-ttu-id="03bc6-121">`x:Array` è che più destinato generale di estendibilità del linguaggio XAML.</span><span class="sxs-lookup"><span data-stu-id="03bc6-121">`x:Array` is more intended for general language extensibility in XAML.</span></span> <span data-ttu-id="03bc6-122">Ma `x:Array` può anche essere utile per specificare i valori di determinate proprietà che accettano le raccolte supportate da XAML come contenuto della proprietà strutturato XAML.</span><span class="sxs-lookup"><span data-stu-id="03bc6-122">But `x:Array` can also be useful for specifying XAML values of certain properties that take XAML-supported collections as their structured property content.</span></span> <span data-ttu-id="03bc6-123">Ad esempio, è possibile specificare il contenuto di un <xref:System.Collections.IEnumerable> proprietà con un `x:Array` utilizzo.</span><span class="sxs-lookup"><span data-stu-id="03bc6-123">For example, you could specify the contents of an <xref:System.Collections.IEnumerable> property with an `x:Array` usage.</span></span>  
  
 <span data-ttu-id="03bc6-124">`x:Array` è un'estensione di markup.</span><span class="sxs-lookup"><span data-stu-id="03bc6-124">`x:Array` is a markup extension.</span></span> <span data-ttu-id="03bc6-125">Le estensioni di markup in genere vengono implementate quando per i valori dell'attributo devono essere utilizzati caratteri escape in modo che non vengano considerati come valori letterali o nomi di gestori e il requisito è più globale del semplice utilizzo di convertitori dei tipi su alcuni tipi o proprietà.</span><span class="sxs-lookup"><span data-stu-id="03bc6-125">Markup extensions are typically implemented when there is a requirement to escape attribute values to be other than literal values or handler names, and the requirement is more global than just putting type converters on certain types or properties.</span></span> <span data-ttu-id="03bc6-126">`x:Array` infatti parzialmente un'eccezione alla regola invece di fornire la gestione dei valori di attributo alternativo, `x:Array` fornisce la gestione alternativa del contenuto di testo interno.</span><span class="sxs-lookup"><span data-stu-id="03bc6-126">`x:Array` is partially an exception to that rule because instead of providing alternative attribute value handling, `x:Array` provides alternative handling of its inner text content.</span></span> <span data-ttu-id="03bc6-127">Questo comportamento consente a tipi che potrebbero non essere supportati da un modello di contenuto esistente per essere raggruppate in una matrice e fa riferimento in un secondo momento nel code-behind accesso all'array denominato;  <span data-ttu-id="03bc6-128">Tutte le estensioni di markup in XAML usano le parentesi graffe ({,} `)` nella sintassi degli attributi, che è la convenzione mediante il quale un processore XAML riconosce che il valore dell'attributo deve essere elaborato da un'estensione di markup.</span><span class="sxs-lookup"><span data-stu-id="03bc6-128">All markup extensions in XAML use the braces ({,}`)` in their attribute syntax, which is the convention by which a XAML processor recognizes that a markup extension must process the attribute value.</span></span> <span data-ttu-id="03bc6-129">Per altre informazioni sulle estensioni di markup in generale, vedere [Type Converters and Markup Extensions for XAML](type-converters-and-markup-extensions-for-xaml.md).</span><span class="sxs-lookup"><span data-stu-id="03bc6-129">For more information about markup extensions in general, see [Type Converters and Markup Extensions for XAML](type-converters-and-markup-extensions-for-xaml.md).</span></span>  
  
 <span data-ttu-id="03bc6-130">XAML 2009, `x:Array` viene definito come un tipo primitivo invece di un'estensione di markup del linguaggio.</span><span class="sxs-lookup"><span data-stu-id="03bc6-130">In XAML 2009, `x:Array` is defined as a language primitive instead of a markup extension.</span></span> <span data-ttu-id="03bc6-131">Per altre informazioni, vedere [i tipi nativi per primitive del linguaggio XAML comuni](built-in-types-for-common-xaml-language-primitives.md).</span><span class="sxs-lookup"><span data-stu-id="03bc6-131">For more information, see [Built-in Types for Common XAML Language Primitives](built-in-types-for-common-xaml-language-primitives.md).</span></span>  
  
## <a name="wpf-usage-notes"></a><span data-ttu-id="03bc6-132">Note sull'utilizzo WPF</span><span class="sxs-lookup"><span data-stu-id="03bc6-132">WPF Usage Notes</span></span>  
 <span data-ttu-id="03bc6-133">In genere, gli elementi di oggetti che popolano un `x:Array` non sono gli elementi esistenti nel [!INCLUDE[TLA2#tla_winclient](../../../includes/tla2sharptla-winclient-md.md)] dello spazio dei nomi XAML e richiede un mapping del prefisso di uno spazio dei nomi XAML non predefinito.</span><span class="sxs-lookup"><span data-stu-id="03bc6-133">Typically, the object elements that populate an `x:Array` are not elements that exist in the [!INCLUDE[TLA2#tla_winclient](../../../includes/tla2sharptla-winclient-md.md)] XAML namespace, and require a prefix mapping to a non-default XAML namespace.</span></span>  
  
 <span data-ttu-id="03bc6-134">Ad esempio, ecco una semplice matrice di due stringhe, con la `sys` prefisso (e anche `x`) definiti a livello della matrice.</span><span class="sxs-lookup"><span data-stu-id="03bc6-134">For example, the following is a simple array of two strings, with the `sys` prefix (and also `x`) defined at the level of the array.</span></span>

```xaml
         xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"

    <sys:String>World</sys:String>
</x:Array>
```

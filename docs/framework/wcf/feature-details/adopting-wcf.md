# <a name="adopting-windows-communication-foundation"></a><span data-ttu-id="ff1cf-102">Uso di Windows Communication Foundation</span><span class="sxs-lookup"><span data-stu-id="ff1cf-102">Adopting Windows Communication Foundation</span></span>

<span data-ttu-id="ff1cf-103">È possibile scegliere di utilizzare Windows Communication Foundation (WCF) per i nuovi sviluppi, mentre continuando a mantenere le applicazioni esistenti sviluppate tramite ASP.NET.</span><span class="sxs-lookup"><span data-stu-id="ff1cf-103">You can choose to use Windows Communication Foundation (WCF) for new development, while continuing to maintain existing applications developed using ASP.NET.</span></span> <span data-ttu-id="ff1cf-104">Poiché WCF è destinato a essere la scelta più adatta per agevolare la comunicazione con applicazioni create con .NET Framework in qualsiasi scenario, che possa servire come uno strumento standard per la risoluzione di un'ampia gamma di problemi di comunicazione software in modo che in ASP.NET non è possibile.</span><span class="sxs-lookup"><span data-stu-id="ff1cf-104">Because WCF is intended to be the most suitable choice for facilitating communication with applications built with the .NET Framework in any scenario, it can serve as a standard tool for solving a wide variety of software communications problems in a way that ASP.NET cannot.</span></span>

<span data-ttu-id="ff1cf-105">Le nuove applicazioni WCF possono essere distribuite nelle macchine stesse come servizi Web ASP.NET esistenti.</span><span class="sxs-lookup"><span data-stu-id="ff1cf-105">New WCF applications can be deployed on the same machines as existing ASP.NET Web services.</span></span> <span data-ttu-id="ff1cf-106">Se i servizi Web ASP.NET esistenti utilizzano una versione di .NET Framework precedenti alla versione 2.0, è possibile utilizzare lo strumento di registrazione ASP.NET IIS per distribuire selettivamente .NET Framework 2.0 alle applicazioni di IIS in cui devono essere ospitate le nuove applicazioni WCF.</span><span class="sxs-lookup"><span data-stu-id="ff1cf-106">If the existing ASP.NET Web services use a version of the .NET Framework prior to version 2.0, then you can use the ASP.NET IIS Registration Tool to selectively deploy the .NET Framework 2.0 to IIS applications in which new WCF applications are to be hosted.</span></span> <span data-ttu-id="ff1cf-107">Tale strumento è documentato al [strumento di registrazione ASP.NET IIS (Aspnet_regiis.exe)](https://go.microsoft.com/fwlink/?LinkId=94687), e ha un'interfaccia utente incorporata nella console di gestione IIS 6.0.</span><span class="sxs-lookup"><span data-stu-id="ff1cf-107">That tool is documented at [ASP.NET IIS Registration Tool (Aspnet_regiis.exe)](https://go.microsoft.com/fwlink/?LinkId=94687), and has a user interface built into the IIS 6.0 management console.</span></span>

<span data-ttu-id="ff1cf-108">WCF può essere utilizzato per aggiungere nuove funzionalità per servizi Web ASP.NET esistenti mediante l'aggiunta di servizi WCF configurati per l'esecuzione in modalità di compatibilità ASP.NET per applicazioni di servizio Web ASP.NET esistenti in IIS.</span><span class="sxs-lookup"><span data-stu-id="ff1cf-108">WCF can be used to add new features to existing ASP.NET Web services by adding WCF services configured to run in ASP.NET compatibility mode to existing ASP.NET Web service applications in IIS.</span></span> <span data-ttu-id="ff1cf-109">A causa delle modalità di compatibilità ASP.NET, il codice per i nuovi servizi WCF può accedere e aggiornare le informazioni sullo stato dell'applicazione stesso come il codice ASP.NET preesistente, utilizzando il <xref:System.Web.HttpContext> classe.</span><span class="sxs-lookup"><span data-stu-id="ff1cf-109">Because of ASP.NET compatibility mode, the code for the new WCF services can access and update the same application state information as the pre-existing ASP.NET code, by using the <xref:System.Web.HttpContext> class.</span></span> <span data-ttu-id="ff1cf-110">Le applicazioni possono inoltre condividere le stesse librerie di classi.</span><span class="sxs-lookup"><span data-stu-id="ff1cf-110">The applications can also share the same class libraries.</span></span>

<span data-ttu-id="ff1cf-111">I client WCF possono utilizzare servizi Web ASP.NET.</span><span class="sxs-lookup"><span data-stu-id="ff1cf-111">WCF clients can use ASP.NET Web services.</span></span> <span data-ttu-id="ff1cf-112">I servizi WCF che sono configurati con la <xref:System.ServiceModel.BasicHttpBinding> può essere utilizzato dai client del servizio Web ASP.NET.</span><span class="sxs-lookup"><span data-stu-id="ff1cf-112">WCF services that are configured with the <xref:System.ServiceModel.BasicHttpBinding> can be used by ASP.NET Web service clients.</span></span> <span data-ttu-id="ff1cf-113">Servizi Web ASP.NET possono coesistere con le applicazioni WCF e WCF può anche essere utilizzato per aggiungere funzionalità a servizi Web ASP.NET esistenti.</span><span class="sxs-lookup"><span data-stu-id="ff1cf-113">ASP.NET Web services can co-exist with WCF applications, and WCF can even be used to add features to existing ASP.NET Web services.</span></span> <span data-ttu-id="ff1cf-114">Tutte le modalità in cui i servizi Web ASP.NET e WCF possono essere usati insieme, è possibile eseguire la migrazione di servizi Web ASP.NET a WCF solo se sono necessarie funzionalità fornite da servizi non Web ASP.NET e WCF.</span><span class="sxs-lookup"><span data-stu-id="ff1cf-114">Given all of these ways in which WCF and ASP.NET Web services can be used together, you may want to migrate ASP.NET Web services to WCF only if you require features that are provided by WCF and not ASP.NET Web services.</span></span>

<span data-ttu-id="ff1cf-115">Anche in alcuni casi in cui è necessario, la migrazione di codice da una tecnologia a un altro raramente è l'approccio corretto.</span><span class="sxs-lookup"><span data-stu-id="ff1cf-115">Even in the few cases where it is necessary, migrating code from one technology to another is seldom the correct approach.</span></span> <span data-ttu-id="ff1cf-116">L'utilizzo della nuova tecnologia è volto a soddisfare requisiti nuovi che non possono essere soddisfatti dalla precedente tecnologia e in tal caso la cosa corretta da fare è progettare una nuova soluzione in grado di soddisfare il nuovo set espanso di requisiti.</span><span class="sxs-lookup"><span data-stu-id="ff1cf-116">The reason for adopting the new technology is to meet new requirements that cannot be met with the earlier technology, and in that case, the correct thing to do is to design a new solution to meet the newly-expanded set of requirements.</span></span> <span data-ttu-id="ff1cf-117">La nuova progettazione si avvale dell'esperienza maturata con il sistema esistente e delle conoscenze acquisite nel frattempo.</span><span class="sxs-lookup"><span data-stu-id="ff1cf-117">The new design benefits from your experience with the existing system and from wisdom gained since that system was designed.</span></span> <span data-ttu-id="ff1cf-118">La nuova progettazione può inoltre utilizzare le funzionalità complete delle nuove tecnologie anziché riprodurre nella nuova piattaforma la progettazione precedente.</span><span class="sxs-lookup"><span data-stu-id="ff1cf-118">The new design can also use the full capabilities of the new technologies rather than reproducing the old design on the new platform.</span></span> <span data-ttu-id="ff1cf-119">Dopo aver creato gli elementi principali della nuova progettazione, diviene più semplice riutilizzare all'interno del nuovo sistema un codice proveniente dal sistema esistente.</span><span class="sxs-lookup"><span data-stu-id="ff1cf-119">After prototyping key elements of the new design, it becomes easier to re-use code from the existing system within the new one.</span></span>

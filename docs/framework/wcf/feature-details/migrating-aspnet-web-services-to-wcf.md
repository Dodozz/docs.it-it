<span data-ttu-id="1e7f4-103">ASP.NET fornisce strumenti e librerie di classi .NET Framework per compilare servizi Web, oltre a funzionalità per servizi di hosting all'interno di Internet Information Services (IIS).</span><span class="sxs-lookup"><span data-stu-id="1e7f4-103">ASP.NET provides .NET Framework class libraries and tools for building Web services, as well as facilities for hosting services within Internet Information Services (IIS).</span></span> <span data-ttu-id="1e7f4-104">Windows Communication Foundation (WCF) offre librerie di classi .NET Framework, strumenti e funzionalità host per consentire a entità software di comunicare utilizzando qualsiasi protocollo, compresi quelli utilizzati dai servizi Web.</span><span class="sxs-lookup"><span data-stu-id="1e7f4-104">Windows Communication Foundation (WCF) provides .NET Framework class libraries, tools and hosting facilities for enabling software entities to communicate using any protocols, including those used by Web services.</span></span>  <span data-ttu-id="1e7f4-105">La migrazione di servizi Web ASP.NET a WCF consente alle applicazioni di sfruttare i vantaggi delle nuove funzionalità e i miglioramenti apportati a WCF.</span><span class="sxs-lookup"><span data-stu-id="1e7f4-105">Migrating ASP.NET Web Services to WCF allows your applications to take advantage of new features and improvements that are unique to WCF.</span></span>  
  
 <span data-ttu-id="1e7f4-106">WCF offre diversi vantaggi importanti rispetto ai servizi Web ASP.NET.</span><span class="sxs-lookup"><span data-stu-id="1e7f4-106">WCF has several important advantages relative to ASP.NET Web services.</span></span> <span data-ttu-id="1e7f4-107">Strumenti per servizi Web ASP.NET sono esclusivamente per la creazione di servizi Web, WCF fornisce strumenti che possono essere utilizzati quando le entità software devono essere eseguite per comunicare tra loro.</span><span class="sxs-lookup"><span data-stu-id="1e7f4-107">While ASP.NET Web services tools are solely for building Web services, WCF provides tools that can be used when software entities must be made to communicate with one another.</span></span> <span data-ttu-id="1e7f4-108">In tal modo, viene ridotto il numero di tecnologie che gli sviluppatori devono conoscere per far fronte a diversi scenari di comunicazione software. Di conseguenza saranno ridotti i costi delle risorse e i tempi di completamento per i progetti di sviluppo del software.</span><span class="sxs-lookup"><span data-stu-id="1e7f4-108">This will reduce the number of technologies that developers are required to know in order to accommodate different software communication scenarios, which in turn will reduce the cost of software development resources, as well as the time to complete software development projects.</span></span>  
  
 <span data-ttu-id="1e7f4-109">Anche per i progetti di sviluppo del servizio Web, WCF supporta più protocolli del servizio Web di supporto dei servizi Web ASP.NET.</span><span class="sxs-lookup"><span data-stu-id="1e7f4-109">Even for Web service development projects, WCF supports more Web service protocols than ASP.NET Web services support.</span></span> <span data-ttu-id="1e7f4-110">I protocolli aggiuntivi sono in grado di gestire soluzioni più sofisticate, tra cui sessioni e transazioni affidabili.</span><span class="sxs-lookup"><span data-stu-id="1e7f4-110">These additional protocols provide for more sophisticated solutions involving, amongst other things, reliable sessions and transactions.</span></span>  
  
 <span data-ttu-id="1e7f4-111">WCF supporta più protocolli di trasporto dei messaggi a servizi Web ASP.NET.</span><span class="sxs-lookup"><span data-stu-id="1e7f4-111">WCF supports more protocols for transporting messages than ASP.NET Web services.</span></span> <span data-ttu-id="1e7f4-112">I servizi Web ASP.NET supportano solo l'invio di messaggi utilizzando HTTP (Hypertext Transfer Protocol).</span><span class="sxs-lookup"><span data-stu-id="1e7f4-112">ASP.NET Web services only support sending messages by using the Hypertext Transfer Protocol (HTTP).</span></span> <span data-ttu-id="1e7f4-113">WCF supporta l'invio di messaggi tramite HTTP, nonché il protocollo TCP (Transmission Control), named pipe e Microsoft Message Queuing (MSMQ).</span><span class="sxs-lookup"><span data-stu-id="1e7f4-113">WCF supports sending messages by using HTTP, as well as the Transmission Control Protocol (TCP), named pipes, and Microsoft Message Queuing (MSMQ).</span></span> <span data-ttu-id="1e7f4-114">Più importante, WCF può essere estesa per supportare protocolli di trasporto aggiuntivi.</span><span class="sxs-lookup"><span data-stu-id="1e7f4-114">More important, WCF can be extended to support additional transport protocols.</span></span> <span data-ttu-id="1e7f4-115">Pertanto, il software sviluppato utilizzando WCF può essere adattato per operare insieme a un'ampia gamma di altri software, aumentando in tal modo il potenziale rendimento dell'investimento.</span><span class="sxs-lookup"><span data-stu-id="1e7f4-115">Therefore, software developed using WCF can be adapted to work together with a wider variety of other software, thereby increasing the potential return on the investment.</span></span>  
  
 <span data-ttu-id="1e7f4-116">WCF fornisce funzionalità molto più ricche per la distribuzione e la gestione delle applicazioni di servizi Web ASP.NET.</span><span class="sxs-lookup"><span data-stu-id="1e7f4-116">WCF provides much richer facilities for deploying and managing applications than ASP.NET Web services provides.</span></span> <span data-ttu-id="1e7f4-117">Oltre a un sistema di configurazione, che è dotato anche ASP.NET, WCF offre un editor di configurazione, traccia delle attività dai mittenti ai destinatari e viceversa attraverso qualsiasi numero di intermediari, un visualizzatore di tracce, la registrazione dei messaggi, un elevato numero di contatori delle prestazioni, e supporto per la Strumentazione gestione Windows.</span><span class="sxs-lookup"><span data-stu-id="1e7f4-117">In addition to a configuration system, which ASP.NET also has, WCF offers a configuration editor, activity tracing from senders to receivers and back through any number of intermediaries, a trace viewer, message logging, a vast number of performance counters, and support for Windows Management Instrumentation.</span></span>  
  
 <span data-ttu-id="1e7f4-118">Visti questi vantaggi potenziali di WCF rispetto all'ASP.NET Web services, se si usa o si prevede di utilizzare servizi Web ASP.NET sono disponibili diverse opzioni:</span><span class="sxs-lookup"><span data-stu-id="1e7f4-118">Given these potential benefits of WCF relative to ASP.NET Web services, if you are using, or are considering using ASP.NET Web services you have several options:</span></span>  
  
-   <span data-ttu-id="1e7f4-119">Continuare a usare i servizi Web ASP.NET e rinunciare ai vantaggi offerti da WCF.</span><span class="sxs-lookup"><span data-stu-id="1e7f4-119">Continue to use ASP.NET Web services, and forego the benefits proffered by WCF.</span></span>  
  
-   <span data-ttu-id="1e7f4-120">Continuare a usare i servizi Web ASP.NET con l'intenzione di adottare WCF in un momento in futuro.</span><span class="sxs-lookup"><span data-stu-id="1e7f4-120">Keep using ASP.NET Web services with the intention of adopting WCF at some time in the future.</span></span> <span data-ttu-id="1e7f4-121">Gli argomenti di questa sezione illustrano come ottimizzare le prospettive per poter utilizzare nuove applicazioni di servizio Web ASP.NET con le applicazioni WCF future.</span><span class="sxs-lookup"><span data-stu-id="1e7f4-121">The topics in this section explain how to maximize the prospects for being able to use new ASP.NET Web service applications together with future WCF applications.</span></span> <span data-ttu-id="1e7f4-122">Negli argomenti di questa sezione illustrano anche come creare nuovo ASP.NET Web services in modo da renderlo più semplice eseguirne la migrazione a WCF.</span><span class="sxs-lookup"><span data-stu-id="1e7f4-122">The topics in this section also explain how to build new ASP.NET Web services so as to make it easier to migrate them to WCF.</span></span> <span data-ttu-id="1e7f4-123">Tuttavia, se è importante proteggere i servizi oppure sono richieste garanzie di affidabilità o transazione, o se personalizzate funzionalità di gestione dovranno essere costruita, quindi è un'opzione migliore per adottare WCF.</span><span class="sxs-lookup"><span data-stu-id="1e7f4-123">However, if securing the services is important, or reliability or transaction assurances are required, or if custom management facilities will have to be constructed, then it is a better option to adopt WCF.</span></span> <span data-ttu-id="1e7f4-124">WCF è progettato espressamente per questi scenari.</span><span class="sxs-lookup"><span data-stu-id="1e7f4-124">WCF is designed for precisely such scenarios.</span></span>  
  
-   <span data-ttu-id="1e7f4-125">Adottare WCF per i nuovi sviluppi, pur continuando a mantenere le applicazioni di servizio Web ASP.NET esistenti.</span><span class="sxs-lookup"><span data-stu-id="1e7f4-125">Adopt WCF for new development, while continuing to maintain your existing ASP.NET Web service applications.</span></span> <span data-ttu-id="1e7f4-126">Questa è molto probabilmente la scelta ottimale.</span><span class="sxs-lookup"><span data-stu-id="1e7f4-126">This choice is very likely the optimal one.</span></span> <span data-ttu-id="1e7f4-127">Produce i vantaggi di WCF, durante la riserva il costo della modifica delle applicazioni esistenti per utilizzarlo.</span><span class="sxs-lookup"><span data-stu-id="1e7f4-127">It yields the benefits of WCF, while sparing the cost of modifying the existing applications to use it.</span></span> <span data-ttu-id="1e7f4-128">In questo scenario, le nuove applicazioni WCF possono coesistere con le applicazioni ASP.NET esistenti.</span><span class="sxs-lookup"><span data-stu-id="1e7f4-128">In this scenario, new WCF applications can co-exist with existing ASP.NET applications.</span></span> <span data-ttu-id="1e7f4-129">Le nuove applicazioni WCF sarà in grado di utilizzare servizi Web ASP.NET esistenti e WCF può essere utilizzato per programmare nuove funzionalità operative in applicazioni ASP.NET esistenti grazie alla modalità di compatibilità ASP.NET di WCF.</span><span class="sxs-lookup"><span data-stu-id="1e7f4-129">New WCF applications will be able to use existing ASP.NET Web services, and WCF can be used to program new operational capabilities into existing ASP.NET applications by virtue of WCF ASP.NET compatibility mode.</span></span>  
  
-   <span data-ttu-id="1e7f4-130">Adottare WCF ed eseguire la migrazione di applicazioni di servizio Web ASP.NET esistenti a WCF.</span><span class="sxs-lookup"><span data-stu-id="1e7f4-130">Adopt WCF and migrate existing ASP.NET Web service applications to WCF.</span></span> <span data-ttu-id="1e7f4-131">È possibile scegliere questa opzione per migliorare le applicazioni esistenti con le funzionalità fornite da WCF o per riprodurre le funzionalità dei servizi Web ASP.NET esistenti all'interno di nuove, più potenti applicazioni WCF.</span><span class="sxs-lookup"><span data-stu-id="1e7f4-131">You may choose this option to enhance the existing applications with features provided by WCF, or to reproduce the functionality of existing ASP.NET Web services within new, more powerful WCF applications.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="1e7f4-132">Prestare attenzione se un servizio WCF è ospitato da IIS 5.x e ASP.NET sia disinstallato.</span><span class="sxs-lookup"><span data-stu-id="1e7f4-132">Care must be taken if a WCF service is hosted by IIS 5.x and ASP.NET is uninstalled.</span></span> <span data-ttu-id="1e7f4-133">Quando un servizio WCF è ospitato da IIS 5.x, il codice per il servizio può essere richiesto se ASP.NET è disinstallato.</span><span class="sxs-lookup"><span data-stu-id="1e7f4-133">When a WCF service is hosted by IIS 5.x, the code for the service can be requested if ASP.NET is uninstalled.</span></span> <span data-ttu-id="1e7f4-134">Quando ASP.NET è disinstallato in un sistema operativo che esegue IIS 5.x e WCF viene disinstallato, un file con estensione svc viene considerato un file di testo e il contenuto, compreso qualsiasi codice sorgente, viene restituito al richiedente.</span><span class="sxs-lookup"><span data-stu-id="1e7f4-134">When ASP.NET is uninstalled on an operating system that is running IIS 5.x and WCF is uninstalled, a file with the .svc extension is considered a text file and the contents, including any source code, is returned to the requester.</span></span>

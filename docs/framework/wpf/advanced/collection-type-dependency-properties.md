## <a name="implementing-a-collection-type-dependency-property"></a><span data-ttu-id="1b882-104">Implementazione di una proprietà di dipendenza di tipo raccolta</span><span class="sxs-lookup"><span data-stu-id="1b882-104">Implementing a Collection-Type Dependency Property</span></span>  
 <span data-ttu-id="1b882-105">Per una proprietà di dipendenza in generale, il modello di implementazione è seguire è che si definisce una [!INCLUDE[TLA2#tla_clr](../../../../includes/tla2sharptla-clr-md.md)] wrapper della proprietà, in cui tale proprietà è supportata da un <xref:System.Windows.DependencyProperty> identificatore piuttosto che un campo o altro costrutto.</span><span class="sxs-lookup"><span data-stu-id="1b882-105">For a dependency property in general, the implementation pattern that you follow is that you define a [!INCLUDE[TLA2#tla_clr](../../../../includes/tla2sharptla-clr-md.md)] property wrapper, where that property is backed by a <xref:System.Windows.DependencyProperty> identifier rather than a field or other construct.</span></span> <span data-ttu-id="1b882-106">Questo stesso modello viene seguito quando si implementa una proprietà di tipo raccolta.</span><span class="sxs-lookup"><span data-stu-id="1b882-106">You follow this same pattern when you implement a collection-type property.</span></span> <span data-ttu-id="1b882-107">Tuttavia, una proprietà di tipo raccolta introduce alcune complessità, per il modello ogni volta che il tipo di contenuto all'interno della raccolta è a sua volta un <xref:System.Windows.DependencyObject> o <xref:System.Windows.Freezable> classe derivata.</span><span class="sxs-lookup"><span data-stu-id="1b882-107">However, a collection-type property introduces some complexity to the pattern whenever the type that is contained within the collection is itself a <xref:System.Windows.DependencyObject> or <xref:System.Windows.Freezable> derived class.</span></span>

 <span data-ttu-id="1b882-109">Quando si crea una proprietà di dipendenza, non si specifica il valore predefinito della proprietà come valore iniziale del campo.</span><span class="sxs-lookup"><span data-stu-id="1b882-109">When you create a dependency property, you do not specify the property default value as the initial field value.</span></span> <span data-ttu-id="1b882-110">Al contrario, viene specificato il valore predefinito tramite i metadati della proprietà di dipendenza.</span><span class="sxs-lookup"><span data-stu-id="1b882-110">Instead, you specify the default value through the dependency property metadata.</span></span> <span data-ttu-id="1b882-111">Se la proprietà è un tipo di riferimento, il valore predefinito specificato nei metadati della proprietà di dipendenza non è un valore predefinito per istanza, ma un valore predefinito che viene applicato a tutte le istanze del tipo.</span><span class="sxs-lookup"><span data-stu-id="1b882-111">If your property is a reference type, the default value specified in dependency property metadata is not a default value per instance; ## <a name="reporting-binding-value-changes-from-collection-properties"></a><span data-ttu-id="1b882-129">Segnalazione di modifiche dei valori di binding nelle proprietà della raccolta</span><span class="sxs-lookup"><span data-stu-id="1b882-129">Reporting Binding Value Changes from Collection Properties</span></span>  
 <span data-ttu-id="1b882-130">Una proprietà della raccolta che costituisce essa stessa una proprietà di dipendenza non segnala automaticamente le modifiche per le relative sottoproprietà.</span><span class="sxs-lookup"><span data-stu-id="1b882-130">A collection property that is itself a dependency property does not automatically report changes to its subproperties.</span></span> <span data-ttu-id="1b882-131">La creazione di binding all'interno di una raccolta può impedire al binding di segnalare le modifiche, invalidando in tal modo alcuni scenari di data binding.</span><span class="sxs-lookup"><span data-stu-id="1b882-131">If you are creating bindings into a collection, this can prevent the binding from reporting changes, thus invalidating some data binding scenarios.</span></span> <span data-ttu-id="1b882-132">Tuttavia, se si usa il tipo di raccolta <xref:System.Windows.FreezableCollection%601> come tipo di raccolta, quindi modifiche della sottoproprietà apportate agli elementi contenuti nella raccolta vengono segnalate correttamente e l'associazione funziona come previsto.</span><span class="sxs-lookup"><span data-stu-id="1b882-132">However, if you use the collection type <xref:System.Windows.FreezableCollection%601> as your collection type, then subproperty changes to contained elements in the collection are properly reported, and binding works as expected.</span></span>  
  
 <span data-ttu-id="1b882-133">Per abilitare il binding di sottoproprietà in una raccolta di oggetti di dipendenza, creare la proprietà della raccolta come tipo <xref:System.Windows.FreezableCollection%601>, con un vincolo di tipo per la raccolta a qualsiasi <xref:System.Windows.DependencyObject> classe derivata.</span><span class="sxs-lookup"><span data-stu-id="1b882-133">To enable subproperty binding in a dependency object collection, create the collection property as type <xref:System.Windows.FreezableCollection%601>, with a type constraint for that collection to any <xref:System.Windows.DependencyObject> derived class.</span></span>

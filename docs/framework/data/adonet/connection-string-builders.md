## <a name="connection-string-injection-attacks"></a><span data-ttu-id="2d29a-110">Attacchi injection alle stringhe di connessione</span><span class="sxs-lookup"><span data-stu-id="2d29a-110">Connection String Injection Attacks</span></span>  
 <span data-ttu-id="2d29a-111">Un attacco injection alle stringhe di connessione può verificarsi quando si usa la concatenazione dinamica di stringhe per compilare stringhe di connessione basate sull'input dell'utente.</span><span class="sxs-lookup"><span data-stu-id="2d29a-111">A connection string injection attack can occur when dynamic string concatenation is used to build connection strings that are based on user input.</span></span> <span data-ttu-id="2d29a-112">Se la stringa non viene convalidata e il testo o i caratteri dannosi non vengono convertiti in caratteri di escape, un utente non autorizzato potrebbe accedere a dati sensibili o ad altre risorse del server.</span><span class="sxs-lookup"><span data-stu-id="2d29a-112">If the string is not validated and malicious text or characters not escaped, an attacker can potentially access sensitive data or other resources on the server.</span></span> <span data-ttu-id="2d29a-113">Ad esempio, un utente non autorizzato potrebbe eseguire un attacco specificando un punto e virgola e aggiungendo un altro valore.</span><span class="sxs-lookup"><span data-stu-id="2d29a-113">For example, an attacker could mount an attack by supplying a semicolon and appending an additional value.</span></span> <span data-ttu-id="2d29a-114">La stringa di connessione viene analizzata tramite un algoritmo basato sul principio che l'ultima occorrenza ha la priorità, pertanto l'input dannoso viene sostituito a un valore lecito.</span><span class="sxs-lookup"><span data-stu-id="2d29a-114">The connection string is parsed by using a "last one wins" algorithm, and the hostile input is substituted for a legitimate value.</span></span>  
  
 <span data-ttu-id="2d29a-115">Le classi di generatori di stringhe di connessione sono progettate per eliminare la necessità di basarsi su congetture e proteggersi da errori di sintassi e vulnerabilità della sicurezza.</span><span class="sxs-lookup"><span data-stu-id="2d29a-115">The connection string builder classes are designed to eliminate guesswork and protect against syntax errors and security vulnerabilities.</span></span> <span data-ttu-id="2d29a-116">Forniscono metodi e proprietà che corrispondono alle coppie chiave/valore note consentite da ogni provider di dati.</span><span class="sxs-lookup"><span data-stu-id="2d29a-116">They provide methods and properties corresponding to the known key/value pairs permitted by each data provider.</span></span> <span data-ttu-id="2d29a-117">Ogni classe mantiene una raccolta fissa di sinonimi e può essere convertita da un sinonimo al nome di chiave noto.</span><span class="sxs-lookup"><span data-stu-id="2d29a-117">Each class maintains a fixed collection of synonyms and can translate from a synonym to the corresponding well-known key name.</span></span> <span data-ttu-id="2d29a-118">Vengono eseguiti controlli per rilevare coppie chiave/valore valide e le coppie non valide generano un'eccezione.</span><span class="sxs-lookup"><span data-stu-id="2d29a-118">Checks are performed for valid key/value pairs and an invalid pair throws an exception.</span></span> <span data-ttu-id="2d29a-119">Inoltre, i valori inseriti vengono gestiti in modo sicuro.</span><span class="sxs-lookup"><span data-stu-id="2d29a-119">In addition, injected values are handled in a safe manner.</span></span>  
  
 <span data-ttu-id="2d29a-120">Nell'esempio seguente viene illustrata la modalità con cui <xref:System.Data.SqlClient.SqlConnectionStringBuilder> gestisce un valore aggiuntivo inserito per l'impostazione `Initial Catalog`.</span><span class="sxs-lookup"><span data-stu-id="2d29a-120">The following example demonstrates how the <xref:System.Data.SqlClient.SqlConnectionStringBuilder> handles an inserted extra value for the `Initial Catalog` setting.</span></span>


# <a name="navigation-property"></a><span data-ttu-id="11e5c-102">proprietà di navigazione</span><span class="sxs-lookup"><span data-stu-id="11e5c-102">navigation property</span></span>
<span data-ttu-id="11e5c-103">Oggetto *proprietà di navigazione* è una proprietà facoltativa su un [tipo di entità](../../../../docs/framework/data/adonet/entity-type.md) che consente la navigazione da uno [end](../../../../docs/framework/data/adonet/association-end.md) di un [associazione](../../../../docs/framework/data/adonet/association-type.md) a l'altra estremità.</span><span class="sxs-lookup"><span data-stu-id="11e5c-103">A *navigation property* is an optional property on an [entity type](../../../../docs/framework/data/adonet/entity-type.md) that allows for navigation from one [end](../../../../docs/framework/data/adonet/association-end.md) of an [association](../../../../docs/framework/data/adonet/association-type.md) to the other end.</span></span> <span data-ttu-id="11e5c-104">A differenza di altri [proprietà](../../../../docs/framework/data/adonet/property.md), le proprietà di navigazione non contengono dati.</span><span class="sxs-lookup"><span data-stu-id="11e5c-104">Unlike other [properties](../../../../docs/framework/data/adonet/property.md), navigation properties do not carry data.</span></span>  
  
 <span data-ttu-id="11e5c-105">Una definizione di proprietà di navigazione include quanto segue:</span><span class="sxs-lookup"><span data-stu-id="11e5c-105">A navigation property definition includes the following:</span></span>  
  
-   <span data-ttu-id="11e5c-106">Un nome</span><span class="sxs-lookup"><span data-stu-id="11e5c-106">A name.</span></span> <span data-ttu-id="11e5c-107">(obbligatorio).</span><span class="sxs-lookup"><span data-stu-id="11e5c-107">(Required)</span></span>  
  
-   <span data-ttu-id="11e5c-108">L'associazione all'interno della quale naviga</span><span class="sxs-lookup"><span data-stu-id="11e5c-108">The association that it navigates.</span></span> <span data-ttu-id="11e5c-109">(obbligatorio).</span><span class="sxs-lookup"><span data-stu-id="11e5c-109">(Required)</span></span>  
  
-   <span data-ttu-id="11e5c-110">Le entità finali dell'associazione all'interno della quale naviga</span><span class="sxs-lookup"><span data-stu-id="11e5c-110">The ends of the association that it navigates.</span></span> <span data-ttu-id="11e5c-111">(obbligatorio).</span><span class="sxs-lookup"><span data-stu-id="11e5c-111">(Required)</span></span>  
  
 <span data-ttu-id="11e5c-112">Si noti che le proprietà di navigazione sono facoltative in entrambi tipi di entità nelle entità finali di un'associazione.</span><span class="sxs-lookup"><span data-stu-id="11e5c-112">Note that navigation properties are optional on both entity types at the ends of an association.</span></span> <span data-ttu-id="11e5c-113">Se si definisce una proprietà di navigazione su un tipo di entità nell'entità finale di un'associazione, non è necessario definire una proprietà di navigazione sul tipo di entità nell'altra entità finale dell'associazione.</span><span class="sxs-lookup"><span data-stu-id="11e5c-113">If you define a navigation property on one entity type at the end of an association, you do not have to define a navigation property on the entity type at the other end of the association.</span></span>  
  
 <span data-ttu-id="11e5c-114">Il tipo di dati di una proprietà di navigazione è determinato dal [molteplicità](../../../../docs/framework/data/adonet/association-end-multiplicity.md) dell'elemento relativo remoto [finale dell'associazione](../../../../docs/framework/data/adonet/association-end.md).</span><span class="sxs-lookup"><span data-stu-id="11e5c-114">The data type of a navigation property is determined by the [multiplicity](../../../../docs/framework/data/adonet/association-end-multiplicity.md) of its remote [association end](../../../../docs/framework/data/adonet/association-end.md).</span></span> <span data-ttu-id="11e5c-115">Si supponga ad esempio che esista una proprietà di navigazione, `OrdersNavProp`, in un tipo di entità `Customer` e che tale proprietà navighi in un'associazione uno-a-molti tra `Customer` e `Order`.</span><span class="sxs-lookup"><span data-stu-id="11e5c-115">For example, suppose a navigation property, `OrdersNavProp`, exists on a `Customer` entity type and navigates a one-to-many association between `Customer` and `Order`.</span></span> <span data-ttu-id="11e5c-116">Poiché l'entità finale dell'associazione remota per la proprietà di navigazione dispone della molteplicità molti (\*), il relativo tipo di dati è una raccolta (di `Order`).</span><span class="sxs-lookup"><span data-stu-id="11e5c-116">Because the remote association end for the navigation property has multiplicity of many (\*), its data type is a collection (of `Order`).</span></span> <span data-ttu-id="11e5c-117">Analogamente, se esiste una proprietà di navigazione `CustomerNavProp` nel tipo di entità `Order`, il relativo tipo di dati sarebbe `Customer`, perché la molteplicità dell'entità finale remota è uno (1).</span><span class="sxs-lookup"><span data-stu-id="11e5c-117">Similarly, if a navigation property, `CustomerNavProp`, exists on the `Order` entity type, its data type would be `Customer`, because the multiplicity of the remote end is one (1).</span></span>

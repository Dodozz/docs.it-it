## <a name="remarks"></a><span data-ttu-id="9eea6-108">Note</span><span class="sxs-lookup"><span data-stu-id="9eea6-108">Remarks</span></span>

```  
select p, GroupPartition(ol.Quantity) from LOB.OrderLines as ol  
  group by ol.Product as p  
```  
  
 <span data-ttu-id="9eea6-110">Le due query seguenti sono semanticamente uguali:</span><span class="sxs-lookup"><span data-stu-id="9eea6-110">The following two queries are semantically equal:</span></span>  
  
```  
select p, Sum(GroupPartition(ol.Quantity)) from LOB.OrderLines as ol  
  group by ol.Product as p  
select p, Sum(ol.Quantity) from LOB.OrderLines as ol  
  group by ol.Product as p  
```  
  
 <span data-ttu-id="9eea6-111">L'operatore `GROUPPARTITION` può essere usato insieme a funzioni di aggregazione definite dall'utente.</span><span class="sxs-lookup"><span data-stu-id="9eea6-111">The `GROUPPARTITION` operator can be used in conjunction with user-defined aggregate functions.</span></span>  
  
 <span data-ttu-id="9eea6-112">`GROUPPARTITION` è un operatore di aggregazione speciale che mantiene un riferimento al set di input raggruppato.</span><span class="sxs-lookup"><span data-stu-id="9eea6-112">`GROUPPARTITION` is a special aggregate operator that holds a reference to the grouped input set.</span></span> <span data-ttu-id="9eea6-113">Questo riferimento può essere usato ovunque nella query laddove GROUP BY è nell'ambito.</span><span class="sxs-lookup"><span data-stu-id="9eea6-113">This reference can be used anywhere in the query where GROUP BY is in scope.</span></span> <span data-ttu-id="9eea6-114">Ad esempio,</span><span class="sxs-lookup"><span data-stu-id="9eea6-114">For example,</span></span>  
  
```  
select p, GroupPartition(ol.Quantity) from LOB.OrderLines as ol group by ol.Product as p  
```  
  
 <span data-ttu-id="9eea6-115">Se si usa un GROUP BY normale, i risultati del raggruppamento sono nascosti.</span><span class="sxs-lookup"><span data-stu-id="9eea6-115">With a regular GROUP BY, the results of the grouping are hidden.</span></span> <span data-ttu-id="9eea6-116">È possibile usare solo i risultati in una funzione di aggregazione.</span><span class="sxs-lookup"><span data-stu-id="9eea6-116">You can only use the results in an aggregate function.</span></span> <span data-ttu-id="9eea6-117">Per vedere i risultati del raggruppamento, è necessario correlarli al set di input tramite una sottoquery.</span><span class="sxs-lookup"><span data-stu-id="9eea6-117">In order to see the results of the grouping, you have to correlate the results of the grouping and the input set by using a subquery.</span></span> <span data-ttu-id="9eea6-118">Le due query seguenti sono equivalenti:</span><span class="sxs-lookup"><span data-stu-id="9eea6-118">The following two queries are equivalent:</span></span>  
  
```  
select p, (select q from GroupPartition(ol.Quantity) as q) from LOB.OrderLines as ol group by ol.Product as p  
select p, (select ol.Quantity as q from LOB.OrderLines as ol2 where ol2.Product = p) from LOB.OrderLines as ol group by ol.Product as p  
```  
  
 <span data-ttu-id="9eea6-119">Come illustrato nell'esempio, grazie all'operatore di aggregazione GROUPPARTITION diventa più semplice ottenere un riferimento al set di input dopo il raggruppamento.</span><span class="sxs-lookup"><span data-stu-id="9eea6-119">As seen from the example, the GROUPPARTITION aggregate operator makes it easier to get a reference to the input set after the grouping.</span></span>  
  
 <span data-ttu-id="9eea6-120">L'operatore GROUPPARTITION può specificare qualsiasi espressione [!INCLUDE[esql](../../../../../../includes/esql-md.md)] nell'operatore di input quando si usa il parametro `expression` .</span><span class="sxs-lookup"><span data-stu-id="9eea6-120">The GROUPPARTITION operator can specify any [!INCLUDE[esql](../../../../../../includes/esql-md.md)] expression in the operator input when you use the `expression` parameter.</span></span>  
  
 <span data-ttu-id="9eea6-121">Tutte le espressioni di input seguenti alla partizione di gruppo sono valide:</span><span class="sxs-lookup"><span data-stu-id="9eea6-121">For instance all of the following input expressions to the group partition are valid:</span></span>  
  
```  
select groupkey, GroupPartition(b) from {1,2,3} as a inner join {4,5,6} as b on true group by a as groupkey  
select groupkey, GroupPartition(1) from {1,2,3} as a inner join {4,5,6} as b on true group by a as groupkey  
select groupkey, GroupPartition(a + b) from {1,2,3} as a inner join {4,5,6} as b on true group by a as groupkey  
select groupkey, GroupPartition({a + b}) from {1,2,3} as a inner join {4,5,6} as b on true group by a as groupkey  
select groupkey, GroupPartition({42}) from {1,2,3} as a inner join {4,5,6} as b on true group by a as groupkey  
select groupkey, GroupPartition(b > a) from {1,2,3} as a inner join {4,5,6} as b on true group by a as groupkey  
```

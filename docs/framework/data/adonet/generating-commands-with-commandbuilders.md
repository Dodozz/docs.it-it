## <a name="optimistic-concurrency-model-for-updates-and-deletes"></a><span data-ttu-id="274f0-135">Modello di concorrenza ottimistica per aggiornamenti ed eliminazioni</span><span class="sxs-lookup"><span data-stu-id="274f0-135">Optimistic Concurrency Model for Updates and Deletes</span></span>  
 <span data-ttu-id="274f0-136">La logica per la generazione automatica di comandi per le istruzioni UPDATE e DELETE è basata sugli *la concorrenza ottimistica*, vale a dire, i record non vengono time.</span></span> <span data-ttu-id="274f0-137">Poiché un record può essere modificato dopo che è stato restituito dall'istruzione SELECT, ma prima che sia eseguita l'istruzione UPDATE o DELETE, nell'istruzione UPDATE o DELETE generata automaticamente è contenuta una clausola WHERE, tale che una riga viene aggiornata solo se contiene tutti i valori originali e non è stata eliminata dall'origine dati.</span><span class="sxs-lookup"><span data-stu-id="274f0-137">Because a record could have been modified after it was returned from the SELECT statement, but before the UPDATE or deleted from the data source.</span></span> <span data-ttu-id="274f0-138">In questo modo si evita di sovrascrivere i nuovi dati.</span><span class="sxs-lookup"><span data-stu-id="274f0-138">This is  <span data-ttu-id="274f0-140">Se si desidera che le istruzioni UPDATE o DELETE vengano completate a prescindere dai valori originali, è necessario impostare in modo esplicito `UpdateCommand` per il `DataAdapter` e non usare la generazione automatica dei comandi.</span><span class="sxs-lookup"><span data-stu-id="274f0-140">If you want the UPDATE or DELETE to complete regardless of original values, you must explicitly set the `UpdateCommand` for the `DataAdapter` and not rely on automatic command generation.</span></span>

## <a name="using-the-commandbuilder-to-automatically-generate-an-sql-statement"></a><span data-ttu-id="274f0-151">Uso di CommandBuilder per generare automaticamente un'istruzione SQL</span><span class="sxs-lookup"><span data-stu-id="274f0-151">Using the CommandBuilder to Automatically Generate an SQL Statement</span></span>  
 <span data-ttu-id="274f0-152">Per generare automaticamente le istruzioni SQL per un oggetto `DataAdapter`, impostare innanzitutto la proprietà `SelectCommand` di `DataAdapter`. Creare quindi un oggetto `CommandBuilder` e specificare come argomento l'oggetto `DataAdapter` per il quale l'oggetto `CommandBuilder` genererà automaticamente le istruzioni SQL.</span><span class="sxs-lookup"><span data-stu-id="274f0-152">To automatically generate SQL statements for a `DataAdapter`, first set the `SelectCommand` property of the `DataAdapter`, then create a `CommandBuilder` object, and specify as an argument the `DataAdapter` for which the `CommandBuilder` will automatically generate SQL statements.</span></span>  
  
```vb  
' Assumes that connection is a valid SqlConnection object   
' inside of a Using block.  
Dim adapter As SqlDataAdapter = New SqlDataAdapter( _  
  "SELECT * FROM dbo.Customers", connection)  
Dim builder As SqlCommandBuilder = New SqlCommandBuilder(adapter)  
builder.QuotePrefix = "["  
builder.QuoteSuffix = "]"  
```  
  
```csharp

// inside of a using block.

  "SELECT * FROM dbo.Customers", connection);

builder.QuotePrefix = "[";  
builder.QuoteSuffix = "]";  
```

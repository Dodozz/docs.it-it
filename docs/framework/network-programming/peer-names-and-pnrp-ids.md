# <a name="peer-names-and-pnrp-ids"></a><span data-ttu-id="cf3fa-102">Nomi di peer e ID PNRP</span><span class="sxs-lookup"><span data-stu-id="cf3fa-102">Peer Names and PNRP IDs</span></span>
<span data-ttu-id="cf3fa-103">Il nome di peer rappresenta un endpoint per la comunicazione e può identificarsi in un computer, un utente, un gruppo, un servizio o qualsiasi altro elemento associato a un peer che può essere risolto in un indirizzo IPv6.</span><span class="sxs-lookup"><span data-stu-id="cf3fa-103">A Peer Name represents an endpoint for communication, which can be a computer, a user, a group, a service, or anything associated with a Peer that can be resolved to an IPv6 address.</span></span> <span data-ttu-id="cf3fa-104">Il protocollo PNRP (Peer Name Resolution Protocol) accetta il nome di peer statisticamente univoco per la creazione di un ID PNRP, usato per identificare i membri del cloud.</span><span class="sxs-lookup"><span data-stu-id="cf3fa-104">The Peer Name Resolution Protocol (PNRP) takes the statistically unique Peer Name for the creation of a PNRP ID, which is used to identify cloud members.</span></span>  
  
## <a name="peer-names"></a><span data-ttu-id="cf3fa-105">Nomi di peer</span><span class="sxs-lookup"><span data-stu-id="cf3fa-105">Peer Names</span></span>  
 <span data-ttu-id="cf3fa-106">I nomi di peer possono essere registrati come non protetti o protetti.</span><span class="sxs-lookup"><span data-stu-id="cf3fa-106">Peer names can be registered as unsecured or secured.</span></span> <span data-ttu-id="cf3fa-107">I nomi non protetti sono semplicemente stringhe di testo soggette a spoofing, dato che chiunque può registrare un nome non protetto duplicato.</span><span class="sxs-lookup"><span data-stu-id="cf3fa-107">Unsecured names are just text strings that are subject to spoofing, as anyone can register a duplicate unsecured name.</span></span> <span data-ttu-id="cf3fa-108">È consigliabile usare i nomi non protetti in reti private o comunque protette.</span><span class="sxs-lookup"><span data-stu-id="cf3fa-108">Unsecured names are best used in private or otherwise protected networks.</span></span> <span data-ttu-id="cf3fa-109">I nomi protetti sono protetti da un certificato e una firma digitale.</span><span class="sxs-lookup"><span data-stu-id="cf3fa-109">Secured names are protected with a certificate and a digital signature.</span></span> <span data-ttu-id="cf3fa-110">Solo chi li ha pubblicati inizialmente ne potrà dimostrare la proprietà.</span><span class="sxs-lookup"><span data-stu-id="cf3fa-110">Only the original publisher will be able to prove ownership of a secured name.</span></span>  
  
 <span data-ttu-id="cf3fa-111">La combinazione di cloud e ambito rende disponibile un ambiente adeguatamente protetto per i peer che partecipano alle attività PNRP.</span><span class="sxs-lookup"><span data-stu-id="cf3fa-111">The combination of cloud and scope provides a reasonably secure environment for peers that participate in PNRP activity.</span></span> <span data-ttu-id="cf3fa-112">Tuttavia, l'uso di un nome di peer protetto non garantisce la sicurezza a livello generale dell'applicazione di rete.</span><span class="sxs-lookup"><span data-stu-id="cf3fa-112">However, using a secured peer name does not ensure the overall security of the networking application.</span></span> <span data-ttu-id="cf3fa-113">La sicurezza dell'applicazione è dipendente dall'implementazione.</span><span class="sxs-lookup"><span data-stu-id="cf3fa-113">Security of the application is implementation-dependent.</span></span>  
  
 <span data-ttu-id="cf3fa-114">I nomi di peer protetti vengono registrati solo dal proprietario e sono protetti con crittografia a chiave pubblica.</span><span class="sxs-lookup"><span data-stu-id="cf3fa-114">Secured peer names are only registered by their owner and are protected with public key cryptography.</span></span> <span data-ttu-id="cf3fa-115">Un nome di peer protetto viene considerato di proprietà dell'entità peer con la chiave privata corrispondente.</span><span class="sxs-lookup"><span data-stu-id="cf3fa-115">A secured peer name is considered owned by the peer entity having the corresponding private key.</span></span> <span data-ttu-id="cf3fa-116">La proprietà può essere dimostrata tramite l'indirizzo peer certificato (CPA), che viene firmato usando la chiave privata.</span><span class="sxs-lookup"><span data-stu-id="cf3fa-116">Ownership can be proved via the certified peer address (CPA), which is signed using the private key.</span></span> <span data-ttu-id="cf3fa-117">Un utente malintenzionato non può falsificare la proprietà di un nome di peer senza la chiave privata corrispondente.</span><span class="sxs-lookup"><span data-stu-id="cf3fa-117">A malicious user cannot forge ownership of a peer name without the corresponding private key.</span></span>  
  
## <a name="pnrp-ids"></a><span data-ttu-id="cf3fa-118">ID PNRP</span><span class="sxs-lookup"><span data-stu-id="cf3fa-118">PNRP IDs</span></span>  
 <span data-ttu-id="cf3fa-119">![ID PNRP](../../../docs/framework/network-programming/media/fdc9e8a0-4a1c-488d-a019-bc3a1973220c.gif "fdc9e8a0-4a1c-488d-a019-bc3a1973220c")</span><span class="sxs-lookup"><span data-stu-id="cf3fa-119">![PNRP ID](../../../docs/framework/network-programming/media/fdc9e8a0-4a1c-488d-a019-bc3a1973220c.gif "fdc9e8a0-4a1c-488d-a019-bc3a1973220c")</span></span>  
  
 <span data-ttu-id="cf3fa-120">Gli ID PNRP sono composti di:</span><span class="sxs-lookup"><span data-stu-id="cf3fa-120">PNRP IDs are composed of the following:</span></span>  
  
-   <span data-ttu-id="cf3fa-121">I 128 bit di ordine superiore, noti come ID peer-to-peer (P2P) sono un hash di un nome di peer assegnato all'endpoint.</span><span class="sxs-lookup"><span data-stu-id="cf3fa-121">The high-order 128 bits, known as the peer-to-peer (P2P) ID, are a hash of a peer name assigned to the endpoint.</span></span> <span data-ttu-id="cf3fa-122">Il nome del peer ha il seguente formato: *Autorità.Classificatore*.</span><span class="sxs-lookup"><span data-stu-id="cf3fa-122">The peer name has the following format: *Authority.Classifier*.</span></span> <span data-ttu-id="cf3fa-123">Nel caso dei nomi protetti, *Autorità* è l'hash SHA1 (Secure Hash Algorithm 1 ) della chiave pubblica del nome di peer in caratteri esadecimali.</span><span class="sxs-lookup"><span data-stu-id="cf3fa-123">For secured names, *Authority* is the Secure Hash Algorithm 1 (SHA1) hash of the public key of the peer name in hexadecimal characters.</span></span> <span data-ttu-id="cf3fa-124">Per i nomi non protetti, *Autorità* è il singolo carattere: "0".</span><span class="sxs-lookup"><span data-stu-id="cf3fa-124">For unsecured names, the *Authority* is the single character "0".</span></span> <span data-ttu-id="cf3fa-125">*Classificatore* è una stringa che identifica l'applicazione.</span><span class="sxs-lookup"><span data-stu-id="cf3fa-125">*Classifier* is a string that identifies the application.</span></span> <span data-ttu-id="cf3fa-126">Nessun classificatore per nome di peer può essere più lungo di 149 caratteri, compreso il carattere di terminazione `null`.</span><span class="sxs-lookup"><span data-stu-id="cf3fa-126">No peer name classifier can be greater than 149 characters long, including the `null` terminator.</span></span>  
  
-   <span data-ttu-id="cf3fa-127">I 128 bit di ordine inferiore vengono usati per la posizione del servizio, ovvero un numero generato che identifica diverse istanze dello stesso ID P2P nello stesso cloud.</span><span class="sxs-lookup"><span data-stu-id="cf3fa-127">The low-order 128 bits are used for the Service Location, which is a generated number that identifies different instances of the same P2P ID in the same cloud.</span></span>  
  
 <span data-ttu-id="cf3fa-128">La combinazione di ID P2P e posizione del servizio consente la registrazione di più ID PNRP da un unico computer.</span><span class="sxs-lookup"><span data-stu-id="cf3fa-128">This combination of P2P ID and Service Location allows multiple PNRP IDs to be registered from a single computer.</span></span>

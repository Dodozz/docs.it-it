# <a name="iclrruntimehoststart-method"></a>Metodo ICLRRuntimeHost::Start
Consente di inizializzare common language runtime (CLR) in un processo.  
  
## <a name="syntax"></a>Sintassi  
  
```  
HRESULT Start();  
```  
  
## <a name="return-value"></a>Valore restituito  
  
|HRESULT|Descrizione|  
|-------------|-----------------|  
|S_OK|`Start` stato restituito correttamente.|  
|HOST_E_CLRNOTAVAILABLE|CLR non è stato caricato in un processo oppure si trova in uno stato in cui non può eseguire codice gestito o elaborare correttamente la chiamata.|  
|HOST_E_TIMEOUT|Timeout della chiamata.|  
|HOST_E_NOT_OWNER|Il chiamante non possiede il blocco.|  
|HOST_E_ABANDONED|Un evento è stato annullato durante un thread bloccato o fiber è rimasta in attesa su di esso.|  
|E_FAIL|Si è verificato un errore irreversibile sconosciuto. Se un metodo viene restituito E_FAIL, CLR non è più utilizzabile all'interno del processo. Le chiamate successive ai metodi di hosting restituiranno HOST_E_CLRNOTAVAILABLE.|  
  
## <a name="remarks"></a>Note  
 In molti scenari non è necessario chiamare `Start`, perché il runtime verrà inizializzato automaticamente alla prima richiesta per eseguire codice gestito. Tuttavia, è possibile utilizzare `Start` specificare esattamente quando deve essere inizializzato il runtime.
